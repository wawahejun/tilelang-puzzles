# Puzzle 07: Scalar FlashAttention Code Analysis

## Overview

This puzzle introduces how to implement **Scalar FlashAttention** in TileLang, an important transition from Softmax to complete FlashAttention. Through a simplified version, we can understand the core ideas of FlashAttention.

## What is FlashAttention?

FlashAttention is an efficient attention mechanism implementation that reduces memory access to accelerate computation. Standard attention mechanism needs to store the complete attention matrix, while FlashAttention uses block computation and Online Softmax to avoid this overhead.

### Standard Attention Mechanism

$$\text{Attention}(Q, K, V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$$

For complete attention:
- $Q, K, V \in \mathbb{R}^{B \times S \times d}$ (B: batch size, S: sequence length, d: head dimension)
- Need to store $QK^T \in \mathbb{R}^{B \times S \times S}$, memory overhead $O(S^2)$

### Scalar FlashAttention Simplification

For progressive learning, we made the following simplifications:

1. **Remove multi-head attention**: Only single head
2. **Scalar version**: $Q, K, V \in \mathbb{R}^{B \times S}$, element-wise multiplication instead of matrix multiplication
3. **Remove scaling factor**: No division by $\sqrt{d_k}$

Simplified computation:

$$O = \text{softmax}(Q \odot K) \odot V$$

where $\odot$ denotes element-wise multiplication.

## Problem Definition

### Input
- `Q: [B, S]` - Query tensor, float32
- `K: [B, S]` - Key tensor, float32
- `V: [B, S]` - Value tensor, float32
- `B: int` - Batch size, 1 ≤ B ≤ 256
- `S: int` - Sequence length, 1 ≤ S ≤ 16384

### Output
- `O: [B, S]` - Output tensor, float32

### Intermediate Variables
- `QK: [B, S]` - Element-wise product of Q and K
- `P: [B, S]` - Softmax result (before normalization)
- `MAX: float32` - Maximum value of each row
- `SUM: float32` - Exponential sum of each row

### Computation Flow

```python
for i in range(B):
    # Step 1: Compute Q*K and find maximum
    MAX = -inf
    for j in range(S):
        QK[i, j] = Q[i, j] * K[i, j]
        MAX = max(QK[i, j], MAX)
    
    # Step 2: Compute exp and sum
    SUM = 0
    for j in range(S):
        P[i, j] = exp(QK[i, j] - MAX)
        SUM += P[i, j]
    
    # Step 3: Normalize and multiply by V
    for j in range(S):
        O[i, j] = P[i, j] / SUM * V[i, j]
```

## PyTorch Reference Implementation

```python
def ref_scalar_flash_attn(Q: torch.Tensor, K: torch.Tensor, V: torch.Tensor):
    return torch.softmax(Q * K, dim=1).mul_(V)
```

Very concise: first compute `Q * K`, then softmax, finally multiply by `V`.

## Relationship with Softmax

Scalar FlashAttention is essentially **Softmax + Additional Computation**:

| Step | Softmax | Scalar FlashAttention |
|------|---------|----------------------|
| Input | A | Q, K, V |
| Preprocessing | - | QK = Q * K |
| Softmax | softmax(A) | softmax(QK) |
| Postprocessing | - | Result * V |

Therefore, we can directly reuse Softmax implementation, just need to:
1. Compute `Q * K` in the first pass
2. Multiply softmax result by `V` in the second pass

## Core Ideas

### 1. Fused Computation

Standard implementation requires 4 steps:
```python
QK = Q * K          # Step 1
P = softmax(QK)     # Step 2 (includes max, exp, sum)
O = P * V           # Step 3
```

FlashAttention fuses these steps, avoiding storing intermediate results `QK` and `P`.

### 2. Online Softmax

Using LSE (log-sum-exp) method, block processing in S dimension:
- First pass: Compute `Q * K`, accumulate LSE simultaneously
- Second pass: Use LSE to compute final output `softmax(QK) * V`

### 3. Memory Efficiency

**Standard Implementation**:
```
Memory: QK[B, S] + P[B, S] + O[B, S] = 3 × B × S
```

**FlashAttention**:
```
Memory: O[B, S] + small block cache = B × S + B × BLOCK_S
```

For S=16384, memory savings are significant.

## Algorithm Flow

```
Input: Q[B, S], K[B, S], V[B, S]
  ↓
┌─────────────────────────────────┐
│ For each batch (parallel process BLOCK_B) │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ First Pass (Compute LSE)         │
│ for s_blk in Serial(S // BLOCK_S): │
│   1. Load Q, K current block     │
│   2. Compute QK = Q * K          │
│   3. Find maximum max(QK)        │
│   4. Compute exp(QK - max)       │
│   5. Sum                         │
│   6. Update LSE                  │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ Second Pass (Compute Output)     │
│ for s_blk in Serial(S // BLOCK_S): │
│   1. Load Q, K, V current block  │
│   2. Compute O = exp(Q*K - LSE) * V │
│   3. Write back output           │
└─────────────────────────────────┘
  ↓
Output: O[B, S]
```

## Differences from Complete FlashAttention

| Feature | Scalar FlashAttention | Complete FlashAttention |
|---------|----------------------|------------------------|
| Dimensions | Q, K, V: [B, S] | Q, K, V: [B, S, d] |
| Computation | Q * K (element-wise) | Q @ K^T (matrix multiplication) |
| Attention Matrix | [B, S] | [B, S, S] |
| Multi-head | No | Yes |
| Scaling | No | Divide by √d |
| Memory Savings | Moderate | Significant (O(S²) → O(S)) |

## Learning Points

1. **Fused Computation**: Fuse multiple operations into one kernel
2. **Online Softmax**: Reuse Softmax's LSE technique
3. **Block Processing**: Serial block processing in S dimension, avoid storing complete matrix
4. **Memory Efficiency**: Only store final output, not intermediate results
5. **Progressive Learning**: Understand core ideas from simplified version, then extend to complete version

## Performance Considerations

Scalar FlashAttention is **both compute and memory bounded**:
- Compute: Q*K, exp, division, multiply by V
- Memory: Need to read Q, K, V multiple times

Optimization strategies:
1. Use Online Softmax to reduce number of passes
2. Block processing to adapt to memory constraints
3. Fused computation to reduce intermediate result storage
4. Batch memory transfers to improve bandwidth utilization
