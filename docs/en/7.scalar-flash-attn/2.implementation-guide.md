# Scalar FlashAttention Implementation Guide

## Core Approach

Scalar FlashAttention = Softmax + Additional Computation:

1. **Preprocessing**: Compute `QK = Q * K`
2. **Softmax**: Apply Online Softmax to `QK` (using LSE)
3. **Postprocessing**: Multiply softmax result by `V`

The key is to fuse these three steps, avoiding storing intermediate results.

---

## Complete Implementation

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_scalar_flash_attn(Q, K, V, BLOCK_B: int, BLOCK_S: int):
    log2_e = 1.44269504
    B, S = T.const("B, S")
    dtype = T.float32
    Q: T.Tensor((B, S), dtype)
    K: T.Tensor((B, S), dtype)
    V: T.Tensor((B, S), dtype)
    O = T.empty((B, S), dtype)

    with T.Kernel(B // BLOCK_B, threads=256) as pid_b:
        # Allocate Fragment
        Q_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        K_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        V_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        O_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        
        # Intermediate results for current block
        cur_QK = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        cur_exp_QK = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        cur_max_QK = T.alloc_fragment((BLOCK_B,), dtype)
        cur_sum_exp_QK = T.alloc_fragment((BLOCK_B,), dtype)
        
        # LSE: Accumulated result across all blocks
        lse = T.alloc_fragment((BLOCK_B,), dtype)
        T.fill(lse, -T.infinity(dtype))
        
        # First pass: Compute LSE (Online Softmax)
        for s_blk_id in T.Serial(S // BLOCK_S):
            # Load current block
            T.copy(Q[pid_b * BLOCK_B, s_blk_id * BLOCK_S], Q_local)
            T.copy(K[pid_b * BLOCK_B, s_blk_id * BLOCK_S], K_local)
            
            # Compute Q * K
            for i, j in T.Parallel(BLOCK_B, BLOCK_S):
                cur_QK[i, j] = Q_local[i, j] * K_local[i, j]
            
            # Find maximum
            T.reduce_max(cur_QK, cur_max_QK, dim=1, clear=True)
            
            # Compute exp(QK - max)
            for i, j in T.Parallel(BLOCK_B, BLOCK_S):
                cur_exp_QK[i, j] = T.exp2(cur_QK[i, j] * log2_e - cur_max_QK[i] * log2_e)
            
            # Sum
            T.reduce_sum(cur_exp_QK, cur_sum_exp_QK, dim=1, clear=True)
            
            # Update LSE
            for i in T.Parallel(BLOCK_B):
                lse[i] = cur_max_QK[i] * log2_e + T.log2(
                    T.exp2(lse[i] - cur_max_QK[i] * log2_e) + cur_sum_exp_QK[i]
                )
        
        # Second pass: Compute final output
        for s_blk_id in T.Serial(S // BLOCK_S):
            T.copy(Q[pid_b * BLOCK_B, s_blk_id * BLOCK_S], Q_local)
            T.copy(K[pid_b * BLOCK_B, s_blk_id * BLOCK_S], K_local)
            T.copy(V[pid_b * BLOCK_B, s_blk_id * BLOCK_S], V_local)
            
            # Compute O = softmax(Q*K) * V = exp(Q*K - LSE) * V
            for i, j in T.Parallel(BLOCK_B, BLOCK_S):
                O_local[i, j] = (
                    T.exp2(Q_local[i, j] * K_local[i, j] * log2_e - lse[i]) * V_local[i, j]
                )
            
            T.copy(O_local, O[pid_b * BLOCK_B, s_blk_id * BLOCK_S])

    return O
```

## Key Techniques Explained

### 1. Comparison with Softmax

| Step | Softmax | Scalar FlashAttention |
|------|---------|----------------------|
| Input Loading | A | Q, K |
| Preprocessing | - | QK = Q * K |
| Find Maximum | max(A) | max(QK) |
| Compute exp | exp(A - max) | exp(QK - max) |
| Sum | sum(exp) | sum(exp) |
| Update LSE | ✓ | ✓ |
| Normalize | / sum | / sum * V |
| Output | softmax(A) | softmax(QK) * V |

**Core Differences**:
1. First pass adds `Q * K` computation
2. Second pass adds multiplication by `V`

### 2. First Pass: Compute LSE

```python
for s_blk_id in T.Serial(S // BLOCK_S):
    # 1. Load Q and K
    T.copy(Q[...], Q_local)
    T.copy(K[...], K_local)
    
    # 2. Compute Q * K (new step)
    for i, j in T.Parallel(BLOCK_B, BLOCK_S):
        cur_QK[i, j] = Q_local[i, j] * K_local[i, j]
    
    # 3-6. Same as Softmax: max, exp, sum, update LSE
    T.reduce_max(cur_QK, cur_max_QK, dim=1, clear=True)
    ...
```

**Key Points**:
- After computing `Q * K`, subsequent steps are exactly the same as Softmax
- Use `cur_QK` to store `Q * K` result
- LSE update formula is consistent with Softmax

### 3. Second Pass: Compute Output

```python
for s_blk_id in T.Serial(S // BLOCK_S):
    # 1. Load Q, K, V (V is new)
    T.copy(Q[...], Q_local)
    T.copy(K[...], K_local)
    T.copy(V[...], V_local)  # New
    
    # 2. Compute O = exp(Q*K - LSE) * V
    for i, j in T.Parallel(BLOCK_B, BLOCK_S):
        O_local[i, j] = (
            T.exp2(Q_local[i, j] * K_local[i, j] * log2_e - lse[i])  # softmax
            * V_local[i, j]  # Multiply by V (new)
        )
    
    # 3. Write back output
    T.copy(O_local, O[...])
```

**Key Points**:
- Need to load V
- Multiply by V while computing softmax
- Fused two operations: `softmax(QK)` and `* V`

### 4. Advantages of Fused Computation

**Non-fused Version**:
```python
# Need to store intermediate results
QK = Q * K              # Store QK[B, S]
P = softmax(QK)         # Store P[B, S]
O = P * V               # Store O[B, S]
# Total memory: 3 × B × S
```

**Fused Version**:
```python
# Only store final output
for each block:
    QK_block = Q_block * K_block  # Temporary, in Fragment
    P_block = softmax(QK_block)   # Temporary, in Fragment
    O_block = P_block * V_block   # Write back to global memory
# Total memory: B × S + B × BLOCK_S (temporary)
```

## Code Structure Summary

```
Kernel(B // BLOCK_B)
├── Allocate Fragment
│   ├── Q_local, K_local, V_local, O_local: Store current block data
│   ├── cur_QK: Q * K result
│   ├── cur_exp_QK, cur_max_QK, cur_sum_exp_QK: Current block intermediate results
│   └── lse: LSE accumulated across all blocks
│
├── First Pass (Compute LSE)
│   └── for s_blk_id in T.Serial(S // BLOCK_S):
│       ├── Load Q_local, K_local
│       ├── Compute cur_QK = Q * K
│       ├── T.reduce_max find maximum
│       ├── Compute exp(QK - max)
│       ├── T.reduce_sum sum
│       └── Update LSE
│
└── Second Pass (Compute Output)
    └── for s_blk_id in T.Serial(S // BLOCK_S):
        ├── Load Q_local, K_local, V_local
        ├── Compute O = exp(Q*K - LSE) * V
        └── Write back O_local
```

## Comparison with Softmax Implementation

### Softmax Implementation

```python
# First pass
T.copy(A[...], A_local)
T.reduce_max(A_local, cur_max, dim=1, clear=True)
# Compute exp and update LSE
...

# Second pass
T.copy(A[...], A_local)
B_local[i, j] = T.exp2(A_local[i, j] * log2_e - lse[i])
T.copy(B_local, B[...])
```

### Scalar FlashAttention Implementation

```python
# First pass
T.copy(Q[...], Q_local)
T.copy(K[...], K_local)
cur_QK[i, j] = Q_local[i, j] * K_local[i, j]  # New
T.reduce_max(cur_QK, cur_max, dim=1, clear=True)
# Compute exp and update LSE (same)
...

# Second pass
T.copy(Q[...], Q_local)
T.copy(K[...], K_local)
T.copy(V[...], V_local)  # New
O_local[i, j] = T.exp2(Q_local[i, j] * K_local[i, j] * log2_e - lse[i]) * V_local[i, j]  # Fused
T.copy(O_local, O[...])
```

**Difference Summary**:
1. Input changes from 1 to 3 (Q, K, V)
2. First pass adds `Q * K` computation
3. Second pass adds multiplication by `V`

## Performance Analysis

### Memory Access

| Operation | Memory Read | Memory Write |
|-----------|-------------|--------------|
| First pass | 2 × B × S (Q, K) | 0 |
| Second pass | 3 × B × S (Q, K, V) | 1 × B × S (O) |
| Total | 5 × B × S | 1 × B × S |

**Comparison with Standard Implementation**:
- Standard: Read 3 × B × S, Write 3 × B × S (QK, P, O)
- FlashAttention: Read 5 × B × S, Write 1 × B × S
- Write reduced by 67%, but read increased (because need to re-read Q, K)

### Computational Complexity

- Q * K: B × S multiplications
- Softmax: B × S exp, max, sum
- Multiply by V: B × S multiplications
- Total: O(B × S)
