# Puzzle 03: Outer Vector Add Code Analysis

## Overview

This puzzle introduces how to implement **outer vector addition** operations in TileLang, moving from the 1D world to the 2D world.

## What is Outer Vector Addition?

Outer Vector Add is an operation that combines two 1D vectors into a 2D matrix:

```
A: [N,]  vector
B: [M,]  vector
C: [N, M]  matrix

C[i, j] = A[i] + B[j]
```

### Intuitive Understanding

Assume `A = [1, 2, 3]`, `B = [10, 20]`:

```
        B[0]=10    B[1]=20
A[0]=1    11        21
A[1]=2    12        22
A[2]=3    13        23
```

Each matrix element `C[i,j]` is the sum of `A[i]` and `B[j]`.

### PyTorch Reference Implementation

```python
def ref_outer_add(A: torch.Tensor, B: torch.Tensor):
    return torch.add(input=A[:, None], other=B[None, :])
```

Using broadcasting: `A[:, None]` becomes `[N, 1]`, `B[None, :]` becomes `[1, M]`, after addition becomes `[N, M]`.

## Core Concepts

### 1. 2D Tensor Processing

Unlike previous puzzles, the output `C` here is a 2D tensor:

```python
C = T.empty((N, M), dtype)  # 2D output
```

### 2. Double Loop Iteration

Need two iterators to traverse A and B respectively:

```python
for i in range(N):      # Traverse A
    for j in range(M):  # Traverse B
        C[i, j] = A[i] + B[j]
```

### 3. Block Partitioning Strategy

Since the output is a 2D matrix, need to consider how to partition blocks:

```python
with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (bx, by):
    # bx: Which block of A to process
    # by: Which block of B to process
```

## Differences from Previous Puzzle

| Feature | Puzzle 02 (Vector Add) | Puzzle 03 (Outer Add) |
|---------|------------------------|----------------------|
| Input dimensions | Two 1D vectors | Two 1D vectors |
| Output dimensions | 1D vector | 2D matrix |
| Computation pattern | Element-wise: `C[i] = A[i] + B[i]` | Outer product style: `C[i,j] = A[i] + B[j]` |
| Block configuration | 1D: `T.Kernel(N // BLOCK_N)` | 2D: `T.Kernel(N // BLOCK_N, M // BLOCK_M)` |
| Index calculation | Single index `base_idx + i` | Double index `(i, j)` |

## Learning Points

1. **2D Kernel Launch**: `T.Kernel(grid_x, grid_y)` supports 2D grid
2. **Multiple Block Indices**: `(bx, by)` represent block indices in two dimensions
3. **Slice Operations**: `A[start:end]` and `C[i_start:i_end, j_start:j_end]`
4. **Data Reuse**: A's elements are reused in each row, B's elements are reused in each column
