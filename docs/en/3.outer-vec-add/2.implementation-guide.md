# Outer Vector Add Implementation Guide

## Task: Implement tl_outer_add

### Basic Implementation (Direct Global Memory Access)

```python
@tilelang.jit
def tl_outer_add(A, B, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float16
    A: T.Tensor((N,), dtype)
    B: T.Tensor((M,), dtype)
    C = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (pid_n, pid_m):
        n_idx = pid_n * BLOCK_N
        m_idx = pid_m * BLOCK_M

        for i in T.Parallel(BLOCK_N):
            for j in T.Parallel(BLOCK_M):
                C[n_idx + i, m_idx + j] = A[n_idx + i] + B[m_idx + j]

    return C
```

### Key Points

1. **2D Kernel Launch**
   ```python
   with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (pid_n, pid_m):
   ```
   - First parameter: Number of blocks in N direction
   - Second parameter: Number of blocks in M direction
   - `(pid_n, pid_m)`: Current block's coordinates in 2D grid

2. **Double Parallel Loop**
   ```python
   for i in T.Parallel(BLOCK_N):
       for j in T.Parallel(BLOCK_M):
   ```
   - Nested `T.Parallel` achieves 2D parallelism
   - Each block processes `BLOCK_N × BLOCK_M` output elements

3. **Index Calculation**
   ```python
   n_idx = pid_n * BLOCK_N  # Start index in N dimension for current block
   m_idx = pid_m * BLOCK_M  # Start index in M dimension for current block
   ```

## Optimized Version (Using Fragment Cache)

The basic version reads `A` and `B` from global memory for each computation, causing repeated access. The optimized version uses `T.copy` to cache data in registers:

```python
@tilelang.jit
def tl_outer_add(A, B, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float16
    A: T.Tensor((N,), dtype)
    B: T.Tensor((M,), dtype)
    C = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (pid_n, pid_m):
        n_idx = pid_n * BLOCK_N
        m_idx = pid_m * BLOCK_M

        # Allocate register fragments
        A_local = T.alloc_fragment((BLOCK_N,), dtype)
        B_local = T.alloc_fragment((BLOCK_M,), dtype)
        C_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)

        # Load data to registers using T.copy
        T.copy(A[n_idx], A_local)
        T.copy(B[m_idx], B_local)

        # Compute in registers
        for i, j in T.Parallel(BLOCK_N, BLOCK_M):
            C_local[i, j] = A_local[i] + B_local[j]

        # Write back to global memory
        T.copy(C_local, C[n_idx, m_idx])

    return C
```

### Optimization Explanation

| Comparison | Basic Version | Optimized Version |
|------------|---------------|-------------------|
| Data access | Read from global memory each time | Preload to registers |
| Memory access count | `BLOCK_N × BLOCK_M × 2` times | `BLOCK_N + BLOCK_M` loads |
| A[i] reuse | Re-read for each j | Cached and reused `BLOCK_M` times |
| B[j] reuse | Re-read for each i | Cached and reused `BLOCK_N` times |

### Data Reuse Diagram

```
Basic Version:
  for i in range(BLOCK_N):       # A[n_idx + i] read BLOCK_M times
      for j in range(BLOCK_M):   # B[m_idx + j] read BLOCK_N times
          C[...] = A[...] + B[...]

Optimized Version:
  T.copy(A[n_idx], A_local)      # A read only once
  T.copy(B[m_idx], B_local)      # B read only once
  for i, j in Parallel:          # All done in registers
      C_local[...] = A_local[...] + B_local[...]
```

## API Summary

| API | Description |
|-----|-------------|
| `T.Kernel(grid_x, grid_y, threads=N)` | 2D grid launch |
| `(pid_n, pid_m)` | 2D block indices |
| `T.Parallel(N)` | Parallel loop (nested use) |
| `T.Parallel(N, M)` | 2D parallel loop (combined syntax) |
| `T.alloc_fragment((N, M), dtype)` | Allocate 2D register fragment |
| `T.copy(src, dst)` | Memory copy |
