# Puzzle 02: Vector Add Code Analysis

## Overview

This puzzle introduces how to implement vector addition operations in TileLang and gradually demonstrates optimization techniques in GPU programming.

## Core Concepts

### 1. Basic Vector Addition (tl_add_1d)

**Goal**: Implement `C[i] = A[i] + B[i]`

```python
@tilelang.jit
def tl_add_1d(A, B, BLOCK_N: int):
    N = T.const("N")
    A: T.Tensor((N,), T.float16)
    B: T.Tensor((N,), T.float16)
    C = T.empty((N,), T.float16)

    with T.Kernel(N // BLOCK_N, threads=256) as bx:
        base_idx = bx * BLOCK_N
        for i in T.Parallel(BLOCK_N):
            C[base_idx + i] = A[base_idx + i] + B[base_idx + i]

    return C
```

**Key Points**:
- `T.Kernel(N // BLOCK_N, threads=256)`: Create N//BLOCK_N blocks, each with 256 threads
- `bx`: Index of current block
- `T.Parallel(BLOCK_N)`: Parallel loop, distributes BLOCK_N elements to 256 threads
- Element-wise operation: `A[i] + B[i]` is scalar addition, not TileOps

### 3. Memory-Optimized Version (tl_mul_relu_1d_mem)

**Optimization Goal**: Reduce global memory accesses, use register optimization

**GPU Memory Hierarchy**:
- **Global Memory (DRAM)**: Slowest, largest capacity
- **Shared Memory**: Medium speed
- **Registers**: Fastest, smallest capacity

**Optimization Strategy**:
1. Use `T.alloc_fragment` to allocate register fragments
2. Batch load data to registers (e.g., use ldg128 to load 128 bits at once)
3. Perform computation in registers
4. Batch write back results

**Fragment Concept**:
- Fragment is TileLang's abstraction of registers
- In CUDA, registers are thread-local
- TileLang's Fragment represents the collection of all threads' registers in a block
- Can operate Fragment uniformly like a Buffer

## Learning Points

1. **Loop Abstraction**: `T.Parallel` for parallel iteration
2. **Element-wise Operations**: Addition, multiplication etc. are scalar operations, not TileOps
3. **Conditional Logic**: `T.if_then_else` for conditional selection
4. **Memory Optimization**: Fragment abstraction simplifies register management
5. **Operator Fusion**: Fuse multiple operations into one kernel to reduce memory access
