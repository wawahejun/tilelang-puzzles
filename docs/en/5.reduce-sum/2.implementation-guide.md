# Reduce Sum Implementation Guide

## Core Approach

Reduce Sum needs to sum all elements in each row:

```python
for i in range(N):
    B[i] = sum(A[i, :])  # Sum row i
```

Key challenges:
1. M dimension is too large, cannot load all at once
2. Need to process in blocks and accumulate results
3. Use `T.reduce_sum` for efficient reduction

---

## Basic Version (Manual Accumulation)

Use element-wise accumulation to understand the essence of reduction:

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_reduce_sum_basic(A, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N,), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as bx:
        n_idx = bx * BLOCK_N

        # Allocate Fragment
        A_frag = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_frag = T.alloc_fragment((BLOCK_N,), dtype)

        # Initialize to 0
        for i in T.Parallel(BLOCK_N):
            B_frag[i] = T.float32(0)

        # Serial traverse all blocks in M dimension
        for m_start in T.Serial(M // BLOCK_M):
            m_idx = m_start * BLOCK_M

            # Load current block
            T.copy(A[n_idx:n_idx+BLOCK_N, m_idx:m_idx+BLOCK_M], A_frag)

            # Accumulate by row
            for i in T.Parallel(BLOCK_N):
                # Use T.alloc_var to create mutable variable
                row_sum = T.alloc_var(dtype)
                row_sum = T.float32(0)
                for j in T.Serial(BLOCK_M):
                    row_sum += A_frag[i, j]
                B_frag[i] += row_sum

        # Write back to global memory
        T.copy(B_frag, B[n_idx:n_idx+BLOCK_N])

    return B
```

**Characteristics**:
- Use nested loops for manual accumulation
- Use `T.alloc_var` to create mutable variables
- Easy to understand the essence of reduction
- But lower efficiency

## Optimized Version (Using T.reduce_sum)

Use TileLang's high-level reduction API:

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_reduce_sum(A, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N,), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
        # Allocate Fragment
        A_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_local = T.alloc_fragment((BLOCK_N,), dtype)

        # Initialize to 0
        T.clear(B_local)

        # Serial traverse all blocks in M dimension
        for m_blk_id in T.Serial(M // BLOCK_M):
            # Load current block
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)

            # Reduce sum and accumulate to B_local
            T.reduce_sum(A_local, B_local, dim=1, clear=False)

        # Write back to global memory
        T.copy(B_local, B[pid_n * BLOCK_N])

    return B
```

## Key Technical Details

### 1. Kernel Configuration

```python
with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
```

- Only block in N dimension
- Each block processes `BLOCK_N` rows
- Total `N // BLOCK_N` blocks

### 2. Initialization

```python
T.clear(B_local)
```

- Use `T.clear` to initialize `B_local` to 0
- More concise than manual loop initialization

### 3. Serial Loop Traversing M Dimension

```python
for m_blk_id in T.Serial(M // BLOCK_M):
    # Process block m_blk_id
```

**Why use `T.Serial`?**
- M dimension is too large, cannot load all at once
- Need to process in blocks and accumulate results
- Serial ensures correct accumulation order

### 4. Key Parameters of T.reduce_sum

```python
T.reduce_sum(A_local, B_local, dim=1, clear=False)
```

**Parameter Explanation**:
- `A_local`: Input Fragment
- `B_local`: Output Fragment (accumulated results)
- `dim=1`: Reduce along second dimension (columns)
- `clear=False`: **Do not clear output**, accumulate to existing value

**Why `clear=False`?**

```python
# First iteration
T.reduce_sum(A_local, B_local, dim=1, clear=False)
# B_local[i] = 0 + sum(A_local[i, :])

# Second iteration
T.reduce_sum(A_local, B_local, dim=1, clear=False)
# B_local[i] = B_local[i] + sum(A_local[i, :])  # Accumulation

# Third iteration...
```

If `clear=True`, `B_local` would be cleared each time, losing previous accumulated results.

**Memory Access Pattern**

```
Global Memory A[N, M]
    ↓ T.copy (batch load BLOCK_M columns)
Fragment A_local[BLOCK_N, BLOCK_M]
    ↓ T.reduce_sum (reduce along dim=1)
Fragment B_local[BLOCK_N] (accumulated)
    ↓ T.copy (batch write back)
Global Memory B[N]
```

## Code Structure Summary

```
Kernel(N // BLOCK_N)
├── Allocate Fragment
│   ├── A_local[BLOCK_N, BLOCK_M]: input block
│   └── B_local[BLOCK_N]: accumulated sum
│
├── Initialization
│   └── T.clear(B_local)  # Clear
│
├── Serial traverse M dimension
│   └── for m_blk_id in T.Serial(M // BLOCK_M):
│       ├── Load A_local
│       └── T.reduce_sum(..., clear=False)  # Accumulate
│
└── Write back results
    └── T.copy(B_local, B[...])
```

## Why Need Serial Loop?

### Problem: M Dimension is Too Large

```
M = 16384, BLOCK_M = 128
Need 16384 / 128 = 128 iterations

Cannot load all 16384 columns to Fragment at once
```

### Solution: Block Accumulation

```
Iteration 0: Load A[:, 0:128]     → B += sum(A[:, 0:128])
Iteration 1: Load A[:, 128:256]   → B += sum(A[:, 128:256])
...
Iteration 127: Load A[:, 16256:16384] → B += sum(A[:, 16256:16384])

Final: B[i] = sum(A[i, :])
```
