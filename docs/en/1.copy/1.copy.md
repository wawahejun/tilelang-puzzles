# Puzzle 01: Copy Code Analysis

## Overview

This is the first puzzle in the TileLang series, introducing the most fundamental operation: **data copy**. Although it seems simple, understanding how `T.copy` works is crucial for subsequent learning.

## Why is Copy So Important?

In GPU programming, data copy is one of the most common operations:
- **Global memory to shared memory**: Load data for computation
- **Shared memory to global memory**: Write back computation results
- **Register-to-register copy**: Intermediate result transfer

Efficient copy operations are the foundation of high-performance GPU kernels.

## Problem Definition

### Part 1: Serial Copy

**Input**:
- `A: [N]` - Input tensor (float16)

**Output**:
- `B: [N]` - Copied tensor (float16)

**Computation Definition**:
```python
for i in range(N):
    B[i] = A[i]
```

### Part 2: Multi-thread Copy

Same problem definition as Part 1, but requires using multiple threads for parallel execution.

### Part 3: Multi-block Copy

**Additional Parameters**:
- `BLOCK_N: int` - Number of elements processed by each thread block

Requires using multiple thread blocks to process different data segments in parallel.

## Core Concepts

### `T.copy` API

`T.copy` is one of the core operations in TileLang, encapsulating efficient data copy logic.

#### Two Indexing Methods

| Method | Syntax | Use Case |
|--------|--------|----------|
| Start index | `T.copy(A[start], local_buf)` | Target is local buffer (auto-inferred) |
| Explicit slice | `T.copy(A[start:end], B[start:end])` | Target is global tensor (must be explicit) |

**Auto-inference Conditions**:
- ✅ Target is local buffer allocated by `T.alloc_fragment` or `T.alloc_shared`
- ❌ Target is global tensor (no shape information, cannot infer)

**Examples**:
```python
# ✅ Auto-inference: Target is local buffer
A_local = T.alloc_fragment((BLOCK_M, BLOCK_K), dtype)
T.copy(A[pid_m * BLOCK_M, k * BLOCK_K], A_local)

# ❌ Cannot auto-infer: Target is global tensor
# T.copy(A[start], B[start])  # Error!

# ✅ Must use explicit slice: Copy between global tensors
T.copy(A[start:end], B[start:end])
```

#### Automatic Optimization

`T.copy` automatically:
1. **Parallelization**: Utilizes all threads within a thread block for parallel copy
2. **Vectorization**: Uses 128-bit vectorized load/store instructions
3. **Memory coalescing**: Optimizes global memory access patterns

### `T.Kernel` Launch Configuration

```python
with T.Kernel(blocks, threads=N) as block_id:
    ...
```

| Parameter | Description |
|-----------|-------------|
| `blocks` | Number of thread blocks |
| `threads` | Number of threads per thread block |
| `block_id` | Index of current thread block (starting from 0) |

### GPU Parallel Hierarchy

```
Grid
├── Block 0 ── Thread 0, Thread 1, ..., Thread N-1
├── Block 1 ── Thread 0, Thread 1, ..., Thread N-1
├── ...
└── Block M-1 ── Thread 0, Thread 1, ..., Thread N-1
```

**Optimization Path**:
```
Serial (1 block, 1 thread)
  ↓  Increase threads within block
Multi-thread (1 block, 128 threads)
  ↓  Increase number of blocks
Multi-block Parallel (M blocks, 128 threads each)
```

## Comparison with PyTorch

| Feature | TileLang `T.copy` | PyTorch `clone()` |
|---------|-------------------|-------------------|
| Execution location | Inside GPU kernel | Host-side call |
| Parallelization | Automatic | Automatic |
| Vectorization | Automatic 128-bit | Automatic |
| Flexibility | Can embed in complex kernels | Standalone operation |
| Use case | Intra-kernel data transfer | Tensor copying |
