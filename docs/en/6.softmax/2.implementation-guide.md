# Softmax Detailed Implementation Analysis

This document provides a complete implementation analysis of the Softmax operator, from basic to optimized versions.

---

## Core Constraints

Softmax requires reduction operations (max and sum) for each row, so the teaching implementation in this document uses the following partitioning:

```
┌────────────────────────────────────────────────────┐
│  Kernel Config: T.Kernel(N // BLOCK_N)             │
│  ──────────────────────────────────────────────    │
│  △ This document does not do cross-block merge     │
│    in M dimension                                   │
│  ✓ Can partition in N dimension (rows independent) │
│  ✓ M dimension uses T.Serial for serial block      │
│    processing                                       │
└────────────────────────────────────────────────────┘
```

The purpose of this approach is not to show that M-dimension parallelism is impossible, but to focus attention on softmax's numerical stability and online update logic.

---

## Version 1: Basic Three-Stage Implementation

Following the standard mathematical definition of Softmax, implement in three stages step by step.

### Mathematical Definition

For input $A \in \mathbb{R}^{N \times M}$, output $B$ is:

$$B_{i,j} = \frac{e^{A_{i,j} - m_i}}{\sum_{k=1}^{M} e^{A_{i,k} - m_i}}$$

where $m_i = \max_j A_{i,j}$ is the maximum value of row $i$.

### Three-Stage Decomposition

For numerical stability, we split the computation into three stages:

```
┌─────────────────────────────────────────────────────────────┐
│  Stage 1: Compute m_i = max_j(A_{i,j})                      │
│  Stage 2: Compute s_i = sum_j(exp(A_{i,j} - m_i))           │
│  Stage 3: Compute B_{i,j} = exp(A_{i,j} - m_i) / s_i        │
└─────────────────────────────────────────────────────────────┘
```

---

### Stage 1: Find Maximum of Each Row

#### Mathematical Formula

$$m_i = \max_{j=1}^{M} A_{i,j}$$

Expanded as loop form:

$$m_i^{(0)} = -\infty$$

$$m_i^{(j+1)} = \max\left(m_i^{(j)}, A_{i,j}\right) \quad \forall j \in [0, M)$$

#### Code Implementation

```python
# m_i^{(0)} = -∞
T.fill(max_val, -T.infinity(dtype))

# m_i^{(j+1)} = max(m_i^{(j)}, A_{i,j})
for m_blk_id in T.Serial(M // BLOCK_M):
    T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
    T.reduce_max(A_local, cur_max, dim=1, clear=True)
    
    for i in T.Parallel(BLOCK_N):
        max_val[i] = T.max(max_val[i], cur_max[i])
```

#### Formula-Code Correspondence

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $m_i^{(0)} = -\infty$ | `T.fill(max_val, -T.infinity(dtype))` |
| $A_{i,j}$ | `A_local[i, j]` |
| $\max_{j}$ | `T.reduce_max(A_local, cur_max, dim=1)` |
| $m_i^{(j+1)} = \max(m_i^{(j)}, A_{i,j})$ | `max_val[i] = T.max(max_val[i], cur_max[i])` |

---

### Stage 2: Compute Exponential Sum

#### Mathematical Formula

$$s_i = \sum_{j=1}^{M} e^{A_{i,j} - m_i}$$

Expanded as loop form:

$$s_i^{(0)} = 0$$

$$s_i^{(j+1)} = s_i^{(j)} + e^{A_{i,j} - m_i} \quad \forall j \in [0, M)$$

#### Code Implementation

```python
# s_i^{(0)} = 0
T.clear(sum_val)

# s_i^{(j+1)} = s_i^{(j)} + e^{A_{i,j} - m_i}
for m_blk_id in T.Serial(M // BLOCK_M):
    T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
    
    # e_{i,j} = e^{A_{i,j} - m_i} = 2^{(A_{i,j} - m_i)·log2(e)}
    for i, j in T.Parallel(BLOCK_N, BLOCK_M):
        B_local[i, j] = T.exp2((A_local[i, j] - max_val[i]) * log2_e)

    # s_i += sum_j(e_{i,j})
    T.reduce_sum(B_local, sum_val, dim=1, clear=False)
```

#### Formula-Code Correspondence

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $s_i^{(0)} = 0$ | `T.clear(sum_val)` |
| $e^{A_{i,j} - m_i}$ | `T.exp2((A_local[i, j] - max_val[i]) * log2_e)` |
| $s_i^{(j+1)} = s_i^{(j)} + e^{A_{i,j} - m_i}$ | `T.reduce_sum(..., clear=False)` |

#### Mathematical Basis for T.exp2
$$e^x = 2^{x\cdot\log_2(e)}$$

where $\log_2(e) \approx 1.44269504$.

```python
# e^{A_{i,j} - m_i} = 2^{log2(e) · (A_{i,j} - m_i)}
T.exp2((A_local[i, j] - max_val[i]) * log2_e)
```

---

### Stage 3: Normalize

#### Mathematical Formula

$$B_{i,j} = \frac{e^{A_{i,j} - m_i}}{s_i}$$

#### Code Implementation

```python
for m_blk_id in T.Serial(M // BLOCK_M):
    T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
    
    # B_{i,j} = e^{A_{i,j} - m_i} / s_i
    for i, j in T.Parallel(BLOCK_N, BLOCK_M):
        B_local[i, j] = T.exp2((A_local[i, j] - max_val[i]) * log2_e)
        B_local[i, j] = B_local[i, j] / sum_val[i]
    
    T.copy(B_local, B[pid_n * BLOCK_N, m_blk_id * BLOCK_M])
```

#### Formula-Code Correspondence

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $e^{A_{i,j} - m_i}$ | `T.exp2((A_local[i, j] - max_val[i]) * log2_e)` |
| $/ s_i$ | `B_local[i, j] / sum_val[i]` |
| $B_{i,j}$ | `B[pid_n * BLOCK_N, m_blk_id * BLOCK_M]` |

### Complete Code

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_softmax_basic(A, BLOCK_N: int, BLOCK_M: int):
    log2_e = 1.44269504
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
        A_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        max_val = T.alloc_fragment((BLOCK_N,), dtype)
        sum_val = T.alloc_fragment((BLOCK_N,), dtype)
        cur_max = T.alloc_fragment((BLOCK_N,), dtype)
        
        # ========== Stage 1: m_i = max_j(A_{i,j}) ==========
        # m_i^{(0)} = -∞
        T.fill(max_val, -T.infinity(dtype))
        
        # m_i^{(j+1)} = max(m_i^{(j)}, A_{i,j})
        for m_blk_id in T.Serial(M // BLOCK_M):
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
            T.reduce_max(A_local, cur_max, dim=1, clear=True)
            for i in T.Parallel(BLOCK_N):
                max_val[i] = T.max(max_val[i], cur_max[i])
        
        # ========== Stage 2: s_i = sum_j(exp(A_{i,j} - m_i)) ==========
        # s_i^{(0)} = 0
        T.clear(sum_val)
        
        # s_i^{(j+1)} = s_i^{(j)} + e^{A_{i,j} - m_i}
        for m_blk_id in T.Serial(M // BLOCK_M):
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
            for i, j in T.Parallel(BLOCK_N, BLOCK_M):
                B_local[i, j] = T.exp2((A_local[i, j] - max_val[i]) * log2_e)
            T.reduce_sum(B_local, sum_val, dim=1, clear=False)
        
        # ========== Stage 3: B_{i,j} = exp(A_{i,j} - m_i) / s_i ==========
        for m_blk_id in T.Serial(M // BLOCK_M):
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
            for i, j in T.Parallel(BLOCK_N, BLOCK_M):
                B_local[i, j] = T.exp2((A_local[i, j] - max_val[i]) * log2_e)
                B_local[i, j] = B_local[i, j] / sum_val[i]
            T.copy(B_local, B[pid_n * BLOCK_N, m_blk_id * BLOCK_M])

    return B
```

## Version 2: Online Softmax (Optimized Version)

### Optimization Motivation

The basic version requires reading A three times, and memory bandwidth is the performance bottleneck. Observation shows Stage 1 and Stage 2 both traverse A, can be merged.

**Core Difficulty**: After merging, $m_i$ changes dynamically, requiring constant adjustment of previously accumulated $s_i$.

---

### LSE (Log-Sum-Exp) Technique

#### Mathematical Definition

$$\text{LSE}(x_1, \ldots, x_n) = \log\left(\sum_{j=1}^{n} e^{x_j}\right)$$

#### Numerically Stable Form

Extract maximum $m = \max_j x_j$:

$$\text{LSE} = m + \log\left(\sum_{j=1}^{n} e^{x_j - m}\right)$$

#### Key Property

$$\text{softmax}(x_i) = e^{x_i - \text{LSE}}$$

**Proof**:

$$e^{x_i - \text{LSE}} = e^{x_i - \log(\sum_j e^{x_j})} = \frac{e^{x_i}}{\sum_j e^{x_j}} = \text{softmax}(x_i)$$

---

### Intra-Block Adjustment Formula (Element-wise)

**Scenario**: When processing a single element, the maximum value changes.

**Mathematical Derivation**:

Let current accumulated exponential sum be $s_{\text{old}}$ (based on maximum $m_{\text{old}}$), new element $A_j$ makes maximum become $m_{\text{new}}$.

Need to adjust:

$$s_{\text{new}} = \sum_{k < j} e^{A_k - m_{\text{new}}} = s_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}}$$

After adding new element:

$$s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}} + e^{A_j - m_{\text{new}}}$$

---

### Inter-Block LSE Update Formula

**Scenario**: Merging LSE results from multiple blocks.

Let:
- $L_{\text{old}}$ = accumulated LSE of all previous blocks
- $m_{\text{cur}}$ = maximum value of current block
- $s_{\text{cur}}$ = exponential sum of current block (already subtracted $m_{\text{cur}}$)

**Mathematical Derivation**:

$$L_{\text{new}} = \log\left(e^{L_{\text{old}}} + \sum_{k \in \text{block}} e^{A_k}\right)$$

Since $\sum_{k \in \text{block}} e^{A_k} = e^{m_{\text{cur}}} \cdot s_{\text{cur}}$:

$$L_{\text{new}} = \log\left(e^{L_{\text{old}}} + e^{m_{\text{cur}}} \cdot s_{\text{cur}}\right)$$

**Formula Consistent with Reference Implementation**: Extract $m_{\text{cur}}$

$$L_{\text{new}} = m_{\text{cur}} + \log\left(e^{L_{\text{old}} - m_{\text{cur}}} + s_{\text{cur}}\right)$$

This is exactly the two-stage LSE merging method used in the reference implementation. Its advantage is short formula, one-to-one correspondence with code:

```python
lse[i] = cur_max_A[i] * log2_e + T.log2(
    T.exp2(lse[i] - cur_max_A[i] * log2_e) + cur_sum_exp_A[i]
)
```

However, avoid a common misunderstanding: this formulation and reference implementation **are not the most mathematically robust stable form**. If current block maximum is significantly smaller than historical block maximum, then `L_old - m_cur` may be positive, and the exponential term may become large. For more robust implementation, generally a stronger anchor selection is used, such as the FlashAttention style formulation.

**Convert to log2 Space**:

Let $\alpha = \log_2(e) \approx 1.44269504$, let $L' = L \cdot \alpha$ (i.e., LSE in $\log_2$ space, corresponding to `lse` variable in code):

$$L'_{\text{new}} = m_{\text{cur}} \cdot \alpha + \log_2\left(2^{L'_{\text{old}} - m_{\text{cur}} \cdot \alpha} + s_{\text{cur}}\right)$$

---

### Alternative Formulation: FlashAttention Style

The LSE method uses logarithmic operations for unified representation. Another intuitive method is to directly maintain global maximum and global exponential sum, which is the core idea of FlashAttention.

#### Mathematical Derivation

Let:
- $m_{\text{global}}$ = global maximum of all processed elements
- $d_{\text{global}}$ = exponential sum of all processed elements (already subtracted $m_{\text{global}}$)

When processing a new block, maximum changes from $m_{\text{old}}$ to $m_{\text{new}}$, need to scale old values:

$$d_{\text{global}}^{\text{new}} = d_{\text{global}}^{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}} + s_{\text{cur}}$$

**Key Property**: Since $m_{\text{old}} \leq m_{\text{new}}$, we have $m_{\text{old}} - m_{\text{new}} \leq 0$, therefore $e^{m_{\text{old}} - m_{\text{new}}} \leq 1$, **no overflow risk**.

Final output:

$$B_{i,j} = \frac{e^{A_{i,j} - m_{\text{global}}}}{d_{\text{global}}}$$

#### Comparison of Two Formulations

| Aspect | LSE Formulation | FlashAttention Formulation |
|--------|-----------------|---------------------------|
| Core Variables | `lse` (single value, consistent with reference answer) | `m_global`, `d_global` (two values) |
| Function Calls | `T.log2()` + `T.exp2()` | `T.exp2()` |
| Temporary Variables | None | Need `m_prev` to save old maximum |
| Code Intuitiveness | Mathematical abstraction | Intuitive and easy to understand |
| Final Output | `exp2(A - lse)` | `exp2(A - m) / d` |


#### Core Code Differences

**LSE Formulation** (update stage):

```python
# Single-step update, using logarithmic operations to merge
for i in T.Parallel(BLOCK_N):
    lse[i] = cur_max_A[i] * log2_e + T.log2(
        T.exp2(lse[i] - cur_max_A[i] * log2_e) + cur_sum_exp_A[i]
    )
```

**FlashAttention Formulation** (update stage):

```python
# Three-step update, no logarithmic operations
for i in T.Parallel(BLOCK_N):
    m_prev = m_global[i]                                    # Save old value
    m_global[i] = T.max(m_global[i], cur_max_A[i])          # Update maximum
    d_global[i] = d_global[i] * T.exp2((m_prev - m_global[i]) * log2_e)  # Scale old sum

# ... compute cur_sum_exp_A for current block ...

for i in T.Parallel(BLOCK_N):
    d_global[i] += cur_sum_exp_A[i]                         # Accumulate new sum
```

**LSE Formulation** (output stage):

```python
for i, j in T.Parallel(BLOCK_N, BLOCK_M):
    B_local[i, j] = T.exp2(A_local[i, j] * log2_e - lse[i])
```

**FlashAttention Formulation** (output stage):

```python
for i, j in T.Parallel(BLOCK_N, BLOCK_M):
    B_local[i, j] = T.exp2((A_local[i, j] - m_global[i]) * log2_e) / d_global[i]
```

#### FlashAttention Style Complete Code

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_softmax_flashattn_style(A, BLOCK_N: int, BLOCK_M: int):
    log2_e = 1.44269504
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
        A_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        cur_exp_A = T.alloc_fragment([BLOCK_N, BLOCK_M], dtype)
        cur_max_A = T.alloc_fragment([BLOCK_N], dtype)
        cur_sum_exp_A = T.alloc_fragment([BLOCK_N], dtype)

        # Separately maintain maximum and exponential sum
        m_global = T.alloc_fragment([BLOCK_N], dtype)  # Global maximum
        d_global = T.alloc_fragment([BLOCK_N], dtype)  # Global exponential sum

        T.fill(m_global, -T.infinity(dtype))
        T.clear(d_global)

        # ========== Pass 1: Online update m_global and d_global ==========
        for m_blk_id in T.Serial(M // BLOCK_M):
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
            T.reduce_max(A_local, cur_max_A, dim=1, clear=True)

            # Step 1: Update m_global, and scale d_global
            for i in T.Parallel(BLOCK_N):
                m_prev = m_global[i]
                m_global[i] = T.max(m_global[i], cur_max_A[i])
                # m_prev - m_global <= 0, safe no overflow
                d_global[i] = d_global[i] * T.exp2((m_prev - m_global[i]) * log2_e)

            # Step 2: Compute exponential of current block relative to global maximum
            for i, j in T.Parallel(BLOCK_N, BLOCK_M):
                cur_exp_A[i, j] = T.exp2((A_local[i, j] - m_global[i]) * log2_e)

            T.reduce_sum(cur_exp_A, cur_sum_exp_A, dim=1, clear=True)

            # Step 3: Accumulate to global sum
            for i in T.Parallel(BLOCK_N):
                d_global[i] += cur_sum_exp_A[i]

        # ========== Pass 2: Compute final output ==========
        for m_blk_id in T.Serial(M // BLOCK_M):
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)

            for i, j in T.Parallel(BLOCK_N, BLOCK_M):
                B_local[i, j] = T.exp2((A_local[i, j] - m_global[i]) * log2_e) / d_global[i]

            T.copy(B_local, B[pid_n * BLOCK_N, m_blk_id * BLOCK_M])

    return B
```

### Complete Code (LSE Formulation)

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_softmax(A, BLOCK_N: int, BLOCK_M: int):
    log2_e = 1.44269504  # α = log_2(e)
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
        A_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        cur_exp_A = T.alloc_fragment([BLOCK_N, BLOCK_M], dtype)
        cur_max_A = T.alloc_fragment([BLOCK_N], dtype)      # m_cur
        cur_sum_exp_A = T.alloc_fragment([BLOCK_N], dtype)  # s_cur
        lse = T.alloc_fragment([BLOCK_N], dtype)            # L_old

        # L^{(0)} = -∞ (because log(0) = -∞)
        T.fill(lse, -T.infinity(dtype))

        # ========== Pass 1: Compute LSE ==========
        for m_blk_id in T.Serial(M // BLOCK_M):
            # Load current block
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
            
            # m_cur = max_j(A_{i,j})
            T.reduce_max(A_local, cur_max_A, dim=1, clear=True)

            # e_{i,j} = e^{A_{i,j} - m_cur} = 2^{log2(e)·A_{i,j} - log2(e)·m_cur}
            for i, j in T.Parallel(BLOCK_N, BLOCK_M):
                cur_exp_A[i, j] = T.exp2(A_local[i, j] * log2_e - cur_max_A[i] * log2_e)

            # s_cur = sum_j(e_{i,j})
            T.reduce_sum(cur_exp_A, cur_sum_exp_A, dim=1, clear=True)

            # lse_new = m_cur·log2(e) + log_2(2^{lse_old - m_cur·log2(e)} + s_cur)
            for i in T.Parallel(BLOCK_N):
                lse[i] = cur_max_A[i] * log2_e + T.log2(
                    T.exp2(lse[i] - cur_max_A[i] * log2_e) + cur_sum_exp_A[i]
                )

        # ========== Pass 2: Compute softmax ==========
        for m_blk_id in T.Serial(M // BLOCK_M):
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)

            # B_{i,j} = 2^{log2(e)·A_{i,j} - lse}   (equivalent to e^{A_{i,j} - LSE}, where lse = log2(e)·LSE)
            for i, j in T.Parallel(BLOCK_N, BLOCK_M):
                B_local[i, j] = T.exp2(A_local[i, j] * log2_e - lse[i])

            T.copy(B_local, B[pid_n * BLOCK_N, m_blk_id * BLOCK_M])

    return B
```

### Detailed Formula-Code Correspondence (LSE Formulation)

#### Pass 1: LSE Computation

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $L'^{(0)} = -\infty$ | `T.fill(lse, -T.infinity(dtype))` |
| $m_{\text{cur}} = \max_j A_{i,j}$ | `T.reduce_max(A_local, cur_max_A, dim=1)` |
| $e_{i,j} = e^{A_{i,j} - m_{\text{cur}}}$ | `T.exp2(A_local[i, j] * log2_e - cur_max_A[i] * log2_e)` |
| $s_{\text{cur}} = \sum_j e_{i,j}$ | `T.reduce_sum(cur_exp_A, cur_sum_exp_A, dim=1)` |
| $L'\_{\text{new}} = m\_{\text{cur}} \alpha + \log\_2(2^{L'\_{\text{old}} - m\_{\text{cur}} \alpha} + s\_{\text{cur}})$ | `lse[i] = cur_max_A[i] * log2_e + T.log2(T.exp2(lse[i] - cur_max_A[i] * log2_e) + cur_sum_exp_A[i])` |

#### Pass 2: Softmax Output

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $B_{i,j} = 2^{\log_2(e) \cdot A_{i,j} - lse}$ | `T.exp2(A_local[i, j] * log2_e - lse[i])` |
| $B_{i,j}$ | `B[pid_n * BLOCK_N, m_blk_id * BLOCK_M]` |

---

### Detailed Formula-Code Correspondence (FlashAttention Formulation)

#### Pass 1: Online Update m_global and d_global

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $m^{(0)} = -\infty$ | `T.fill(m_global, -T.infinity(dtype))` |
| $d^{(0)} = 0$ | `T.clear(d_global)` |
| $m_{\text{cur}} = \max_j A_{i,j}$ | `T.reduce_max(A_local, cur_max_A, dim=1)` |
| $m_{\text{prev}} = m_{\text{old}}$ | `m_prev = m_global[i]` |
| $m_{\text{new}} = \max(m_{\text{old}}, m_{\text{cur}})$ | `m_global[i] = T.max(m_global[i], cur_max_A[i])` |
| $d_{\text{scaled}} = d_{\text{old}} \cdot e^{m_{\text{prev}} - m_{\text{new}}}$ | `d_global[i] = d_global[i] * T.exp2((m_prev - m_global[i]) * log2_e)` |
| $e_{i,j} = e^{A_{i,j} - m_{\text{new}}}$ | `T.exp2((A_local[i, j] - m_global[i]) * log2_e)` |
| $s_{\text{cur}} = \sum_j e_{i,j}$ | `T.reduce_sum(cur_exp_A, cur_sum_exp_A, dim=1)` |
| $d_{\text{new}} = d_{\text{scaled}} + s_{\text{cur}}$ | `d_global[i] += cur_sum_exp_A[i]` |

#### Pass 2: Softmax Output

| Mathematical Formula | Code Implementation |
|---------------------|---------------------|
| $B_{i,j} = e^{A_{i,j} - m_{\text{global}}} / d_{\text{global}}$ | `T.exp2((A_local[i, j] - m_global[i]) * log2_e) / d_global[i]` |
| $B_{i,j}$ | `B[pid_n * BLOCK_N, m_blk_id * BLOCK_M]` |

## Version Comparison

| Aspect | Basic Version | Online Softmax (LSE) | Online Softmax (FlashAttention) |
|--------|--------------|----------------------|--------------------------------|
| Mathematical Form | $m_i$, $s_i$ computed separately | LSE unified representation | $m$, $d$ maintained separately |
| Number of Passes | 3 | 2 | 2 |
| Memory Access | Read A three times | Read A twice | Read A twice |
| Function Calls | `T.exp2` | `T.log2` + `T.exp2` | `T.exp2` |
| Extra Variables | None | None | `m_prev` |
| Code Complexity | Simple | Medium | Medium |
| Intuitiveness | High | Low (mathematical abstraction) | High (direct understanding) |

## Summary

### Learning Path

```
Basic Version: Mathematical formula → Code implementation → One-to-one correspondence
    │
    │  Question: Why three passes?
    │
    ▼
Optimization Motivation: Merge Stage 1 and 2 → max changes dynamically
    │
    │  Solution
    │
    ├─→ LSE Formulation: Mathematically elegant, logarithmic unified representation
    │
    └─→ FlashAttention Formulation: Natural and intuitive
```

### Key Points

| Concept | Mathematics | Code |
|---------|-------------|------|
| Numerical Stability | $e^{x - m}$ where $m = \max(x)$ | `T.exp2((x - max_val) * log2_e)` |
| Maximum Reduction | $m_i = \max_j A_{i,j}$ | `T.reduce_max(..., dim=1)` |
| Exponential Sum | $s_i = \sum_j e^{A_{i,j} - m_i}$ | `T.reduce_sum(..., dim=1)` |
| LSE Update | $lse_{\text{new}} = m_{\text{cur}} \cdot \alpha + \log_2(2^{lse_{\text{old}} - m_{\text{cur}} \cdot \alpha} + s_{\text{cur}})$ | `lse[i] = cur_max_A[i] * log2_e + T.log2(...)` |
| FlashAttention Update | $d_{\text{new}} = d_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}} + s_{\text{cur}}$ | `d_global[i] = d_global[i] * T.exp2((m_prev - m_global[i]) * log2_e) + cur_sum_exp_A[i]` |
| Final Output (LSE) | $B_{i,j} = 2^{\log_2(e) \cdot A_{i,j} - lse}$ | `T.exp2(A_local[i, j] * log2_e - lse[i])` |
| Final Output (FlashAttention) | $B_{i,j} = e^{A_{i,j} - m} / d$ | `T.exp2((A_local[i, j] - m_global[i]) * log2_e) / d_global[i]` |

Additional engineering notes related to reference implementation:

- All complete code in this document assumes `N % BLOCK_N == 0` and `M % BLOCK_M == 0`, omitting tail/mask logic.
- Output buffer remains `float32`, keeping focus on online softmax itself; for mixed precision, can discuss cast separately at final write-back.
