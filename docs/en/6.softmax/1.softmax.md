# Puzzle 06: Softmax Implementation Challenge

## Overview

Softmax is one of the most fundamental operations in neural networks, widely used in classification tasks and attention mechanisms. This puzzle requires you to implement a high-performance Softmax operator in TileLang.

## What is Softmax?

Softmax converts a set of real numbers into a probability distribution:

$$\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_{j=1}^{n} e^{x_j}}$$

For a 2D tensor `[N, M]`, Softmax is typically computed along the last dimension (rows).

## Problem Definition

### Input and Output

```
Input: A: [N, M] (float32)
Output: B: [N, M] (float32)
Constraints: N ∈ [1, 4096], M ∈ [1, 16384]
```
> To focus on softmax itself, the example code assumes `N % BLOCK_N == 0` and `M % BLOCK_M == 0`, without tail handling.

### Computation Flow (Python Pseudocode)

```python
for i in range(N):
    # Step 1: Find the maximum value of each row
    MAX = -inf
    for j in range(M):
        MAX = max(A[i, j], MAX)
    
    # Step 2: Compute exp(x - MAX) and accumulate
    SUM = 0
    for j in range(M):
        B[i, j] = exp(A[i, j] - MAX)
        SUM += B[i, j]
    
    # Step 3: Normalize
    for j in range(M):
        B[i, j] /= SUM
```

## Part 1: Numerical Stability and Basic Implementation

### Numerical Stability: Subtract Maximum

Directly computing $e^{x}$ may cause numerical overflow. When $x = 1000$, $e^{1000}$ far exceeds the representable range of float32 (about $10^{38}$), producing `inf`.

**Solution**: Using the property of the exponential function, multiply both numerator and denominator by $e^{-m}$:

$$\frac{e^{x_i}}{\sum_j e^{x_j}} = \frac{e^{x_i} \cdot e^{-m}}{\sum_j e^{x_j} \cdot e^{-m}} = \frac{e^{x_i - m}}{\sum_j e^{x_j - m}}$$

where $m = \max_j x_j$. After subtracting the maximum, all $x_j - m \leq 0$, so $e^{x_j - m} \in (0, 1]$, numerically stable.

### Reduction Variable Initialization

| Operation | Initial Value | Reason |
|-----------|---------------|--------|
| Find maximum | $-\infty$ | Any number is greater than $-\infty$, ensuring the first element updates correctly |
| Sum | $0$ | Additive identity, doesn't affect accumulated result |

```python
# Find maximum: initial value should be -inf
max_val = T.alloc_fragment((BLOCK_N,), dtype)
T.fill(max_val, -T.infinity(dtype))

# Sum: initial value should be 0
sum_val = T.alloc_fragment((BLOCK_N,), dtype)
T.clear(sum_val)
```

Note: `T.clear` sets to 0, `T.fill` can set any value.

## Part 2: From Three-Stage to Two-Stage Optimization

### Three-Stage Memory Access Analysis

Standard Softmax requires three passes over the M dimension:

```
Pass 1: Find maximum   → Read A once
Pass 2: Compute exp    → Read A once, write B once
Pass 3: Normalize      → Read B once, write B once
```

Memory access is the performance bottleneck. Can we reduce the number of passes?

### Difficulty in Merging Pass 1 and Pass 2

Observe Pass 1 and Pass 2:

```python
# Pass 1
for j in range(M):
    MAX = max(A[i, j], MAX)

# Pass 2  
for j in range(M):
    B[i, j] = exp(A[i, j] - MAX)
    SUM += B[i, j]
```

If merged into one loop, we encounter a problem: during traversal, `MAX` constantly changes.

**Concrete Example**:

```
Processed elements: A₀=3, A₁=4, A₂=5
Current maximum: m_old = 5
Accumulated exponential sum: s_old = e^(3-5) + e^(4-5) + e^(5-5)

Next element: A₃ = 10
New maximum: m_new = 10
```

Problem: $s_{\text{old}}$ was computed based on $m=5$, but now $m$ has become $10$, needs adjustment!

---

### Intra-Block Adjustment Formula (Element-wise)

**Scenario**: Within a single block, processing elements one by one, the maximum value changes.

Let old maximum $m_{\text{old}}$, new maximum $m_{\text{new}}$. Previous exponential sum:

$$s_{\text{old}} = \sum_{k < j} e^{A_k - m_{\text{old}}}$$

Needs to be converted to a form based on $m_{\text{new}}$:

$$s_{\text{new}} = \sum_{k < j} e^{A_k - m_{\text{new}}}$$

**Derivation**: From exponential property

$$e^{A_k - m_{\text{new}}} = e^{A_k - m_{\text{old}}} \cdot e^{m_{\text{old}} - m_{\text{new}}}$$

Thus the adjustment formula is:

$$s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}}$$

After adding the current element:

$$s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}} + e^{A_j - m_{\text{new}}}$$

---

### Block Processing: Intra-Block vs Inter-Block

When $M$ is large, block processing is needed. Understanding two granularities of update formulas is key:

```
┌─────────────────────────────────────────────────────────────┐
│                      Full Data (M is large)                  │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐            │
│  │ Block 0 │ │ Block 1 │ │ Block 2 │ │ Block 3 │            │
│  │ A₀...A₃ │ │ A₄...A₇ │ │ A₈...A₁₁│ │A₁₂...A₁₅│            │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘            │
│       │           │           │           │                  │
│       ▼           ▼           ▼           ▼                  │
│    Intra-block  Intra-block  Intra-block  Intra-block        │
│    (element-wise)(element-wise)(element-wise)(element-wise)  │
│       │           │           │           │                  │
│       ▼           ▼           ▼           ▼                  │
│     LSE₀        LSE₁        LSE₂        LSE₃                 │
│       │           │           │           │                  │
│       └─────┬─────┴─────┬─────┴─────┬─────┘                  │
│             │           │           │                        │
│             ▼           ▼           ▼                        │
│           Inter-block LSE merge (cross-block update formula) │
│             │                                               │
│             ▼                                               │
│         Final LSE                                           │
└─────────────────────────────────────────────────────────────┘
```

**Two Levels**:

| Level | Granularity | Formula | Purpose |
|-------|-------------|---------|---------|
| **Intra-block** | Single element | $s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}}-m_{\text{new}}} + e^{A_j-m_{\text{new}}}$ | Process each element within block |
| **Inter-block** | Entire block | $L_{\text{new}} = m_{\text{cur}} + \log(e^{L_{\text{old}}-m_{\text{cur}}} + s_{\text{cur}})$ | Merge results from multiple blocks |

---

### LSE (Log-Sum-Exp) Representation

To avoid frequent exponential operations, use LSE representation:

$$\text{LSE} = \log\left(\sum_j e^{x_j}\right)$$

**Key Property**:

$$\text{softmax}(x_i) = e^{x_i - \text{LSE}}$$

Numerically stable form of LSE:

$$\text{LSE} = m + \log\left(\sum_j e^{x_j - m}\right)$$

where $m = \max(x_j)$.

---

### Inter-Block LSE Update Formula (Cross-Block Merge)

**Scenario**: Merging LSE results from multiple blocks.

Let:
- $L_{\text{old}}$ = LSE of all previous blocks
- $m_{\text{cur}}$ = maximum value of current block
- $s_{\text{cur}}$ = exponential sum of current block (already subtracted $m_{\text{cur}}$)

**Deriving New LSE**:

From LSE definition, the merged exponential sum is:

$$e^{L_{\text{new}}} = e^{L_{\text{old}}} + \underbrace{e^{m_{\text{cur}}} \cdot s_{\text{cur}}}_{\text{actual exponential sum of current block}}$$

Therefore:

$$L_{\text{new}} = \log\left(e^{L_{\text{old}}} + e^{m_{\text{cur}}} \cdot s_{\text{cur}}\right)$$

**Numerical Stability Handling**: Extract $m_{\text{cur}}$

$$L_{\text{new}} = m_{\text{cur}} + \log\left(e^{L_{\text{old}} - m_{\text{cur}}} + s_{\text{cur}}\right)$$

**Teaching Note**:
- This document aligns with the puzzle reference implementation, directly using current block maximum $m_{\text{cur}}$ as the anchor for merging
- When the current block contains a new global maximum, $L_{\text{old}} - m_{\text{cur}}$ is typically small, this formulation is intuitive and consistent with code
- However, strictly speaking, $L_{\text{old}} - m_{\text{cur}} \leq 0$ **does not always hold**; if historical block maximum is much larger than current block maximum, this term may be positive
- Therefore, this LSE formulation is more suitable as a teaching version consistent with the reference implementation; for the most robust engineering implementation, refer to the FlashAttention style formulation below

**Concrete Example**:

```
Assume:
  - Processed Block 0-1, L_old = 5
  - Block 2 has m_cur = 10, s_cur = 3

Direct calculation:
  L_new = log(e^5 + e^10 × 3) = log(148.4 + 66079.4) ≈ 11.1

Stable calculation:
  L_new = 10 + log(e^(5-10) + 3) = 10 + log(0.0067 + 3) = 10 + 1.1 = 11.1

Same result, but the latter avoids potential overflow risks like e^100!
```

---

### Converting to log2 Space

GPU hardware typically has more direct support for power-of-2 operations, so in teaching and engineering practice, `exp2` is often preferred.

$$\exp(x) = 2^{x \cdot \log_2(e) }$$

Inter-block update formula converted to log2 space:

Let $L' = L \cdot \log_2(e)$, then the formula consistent with the reference implementation can be written as:

$$L'_{\text{new}} = m_{\text{cur}} \cdot \log_2(e) + \log_2\left(2^{L'_{\text{old}} - m_{\text{cur}} \cdot \log_2(e)} + s_{\text{cur}}\right)$$

where $\log_2(e) \approx 1.44269504$.

```python
log2_e = 1.44269504
result = T.exp2(x * log2_e)  # Recommended
```
