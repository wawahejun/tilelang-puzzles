# Puzzle 06: Softmax 实现挑战

## 概述

Softmax 是神经网络中最基础的操作之一，广泛应用于分类任务和注意力机制。这个 puzzle 要求你在 TileLang 中实现高性能的 Softmax 算子。

## 什么是 Softmax?

Softmax 将一组实数转换为概率分布：

$$\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_{j=1}^{n} e^{x_j}}$$

对于 2D 张量 `[N, M]`，通常沿着最后一个维度（行）计算 Softmax。

## 问题定义

### 输入输出

```
输入: A: [N, M] (float32)
输出: B: [N, M] (float32)
约束: N ∈ [1, 4096], M ∈ [1, 16384]
```
> 为了聚焦 softmax 本身，示例代码默认 `N % BLOCK_N == 0` 且 `M % BLOCK_M == 0`，未展开 tail 处理。

### 计算流程（Python 伪代码）

```python
for i in range(N):
    # 步骤1: 找到每行的最大值
    MAX = -inf
    for j in range(M):
        MAX = max(A[i, j], MAX)
    
    # 步骤2: 计算 exp(x - MAX) 并累加
    SUM = 0
    for j in range(M):
        B[i, j] = exp(A[i, j] - MAX)
        SUM += B[i, j]
    
    # 步骤3: 归一化
    for j in range(M):
        B[i, j] /= SUM
```

## 第一部分：数值稳定性与基础实现

### 数值稳定性：减去最大值

直接计算 $e^{x}$ 可能导致数值溢出。当 $x = 1000$ 时，$e^{1000}$ 远超 float32 的表示范围（约 $10^{38}$），会产生 `inf`。

**解决方案**：利用指数函数的性质，在分子分母同时乘以 $e^{-m}$：

$$\frac{e^{x_i}}{\sum_j e^{x_j}} = \frac{e^{x_i} \cdot e^{-m}}{\sum_j e^{x_j} \cdot e^{-m}} = \frac{e^{x_i - m}}{\sum_j e^{x_j - m}}$$

其中 $m = \max_j x_j$。减去最大值后，所有 $x_j - m \leq 0$，因此 $e^{x_j - m} \in (0, 1]$，数值稳定。

### 归约变量初始化

| 操作 | 初始值 | 原因 |
|------|--------|------|
| 找最大值 | $-\infty$ | 任何数都比 $-\infty$ 大，保证第一个元素能正确更新 |
| 求和 | $0$ | 加法单位元，不影响累积结果 |

```python
# 找最大值: 初始值应为 -inf
max_val = T.alloc_fragment((BLOCK_N,), dtype)
T.fill(max_val, -T.infinity(dtype))

# 求和: 初始值应为 0
sum_val = T.alloc_fragment((BLOCK_N,), dtype)
T.clear(sum_val)
```

注意：`T.clear` 设为 0，`T.fill` 可设任意值。

## 第二部分：从三阶段到两阶段的优化

### 三阶段内存访问分析

标准 Softmax 需要三次遍历 M 维度：

```
Pass 1: 找最大值   → 读取 A 一次
Pass 2: 计算 exp   → 读取 A 一次，写入 B 一次
Pass 3: 归一化     → 读取 B 一次，写入 B 一次
```

内存访问是性能瓶颈，能否减少遍历次数？

### 合并 Pass 1 和 Pass 2 的困难

观察 Pass 1 和 Pass 2：

```python
# Pass 1
for j in range(M):
    MAX = max(A[i, j], MAX)

# Pass 2  
for j in range(M):
    B[i, j] = exp(A[i, j] - MAX)
    SUM += B[i, j]
```

如果合并成一个循环，会遇到问题：在遍历过程中，`MAX` 会不断变化。

**具体例子**：

```
已处理元素: A₀=3, A₁=4, A₂=5
当前最大值: m_old = 5
累积指数和: s_old = e^(3-5) + e^(4-5) + e^(5-5)

下一个元素: A₃ = 10
新最大值: m_new = 10
```

问题：$s_{\text{old}}$ 是基于 $m=5$ 计算的，但现在 $m$ 变成了 $10$，需要调整！

---

### 块内调整公式（逐元素）

**场景**：在单个块内，逐元素处理时，最大值发生变化。

设旧最大值 $m_{\text{old}}$，新最大值 $m_{\text{new}}$。之前的指数和：

$$s_{\text{old}} = \sum_{k < j} e^{A_k - m_{\text{old}}}$$

需要转换为基于 $m_{\text{new}}$ 的形式：

$$s_{\text{new}} = \sum_{k < j} e^{A_k - m_{\text{new}}}$$

**推导**：由指数性质

$$e^{A_k - m_{\text{new}}} = e^{A_k - m_{\text{old}}} \cdot e^{m_{\text{old}} - m_{\text{new}}}$$

因此调整公式为：

$$s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}}$$

加上当前元素后：

$$s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}} - m_{\text{new}}} + e^{A_j - m_{\text{new}}}$$

---

### 分块处理：块内 vs 块间

当 $M$ 很大时，需要分块处理。理解两种粒度的更新公式是关键：

```
┌─────────────────────────────────────────────────────────────┐
│                      完整数据 (M 很大)                        │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐            │
│  │ Block 0 │ │ Block 1 │ │ Block 2 │ │ Block 3 │            │
│  │ A₀...A₃ │ │ A₄...A₇ │ │ A₈...A₁₁│ │A₁₂...A₁₅│            │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘            │
│       │           │           │           │                  │
│       ▼           ▼           ▼           ▼                  │
│    块内处理     块内处理     块内处理     块内处理             │
│    (逐元素)    (逐元素)    (逐元素)    (逐元素)               │
│       │           │           │           │                  │
│       ▼           ▼           ▼           ▼                  │
│     LSE₀        LSE₁        LSE₂        LSE₃                 │
│       │           │           │           │                  │
│       └─────┬─────┴─────┬─────┴─────┬─────┘                  │
│             │           │           │                        │
│             ▼           ▼           ▼                        │
│           块间 LSE 合并（跨块更新公式）                        │
│             │                                               │
│             ▼                                               │
│         最终 LSE                                            │
└─────────────────────────────────────────────────────────────┘
```

**两个层次**：

| 层次 | 粒度 | 公式 | 目的 |
|------|------|------|------|
| **块内** | 单个元素 | $s_{\text{new}} = s_{\text{old}} \cdot e^{m_{\text{old}}-m_{\text{new}}} + e^{A_j-m_{\text{new}}}$ | 处理块内每个元素 |
| **块间** | 整个块 | $L_{\text{new}} = m_{\text{cur}} + \log(e^{L_{\text{old}}-m_{\text{cur}}} + s_{\text{cur}})$ | 合并多个块的结果 |

---

### LSE (Log-Sum-Exp) 表示

为了避免频繁的指数运算，使用 LSE 表示：

$$\text{LSE} = \log\left(\sum_j e^{x_j}\right)$$

**关键性质**：

$$\text{softmax}(x_i) = e^{x_i - \text{LSE}}$$

LSE 的数值稳定形式：

$$\text{LSE} = m + \log\left(\sum_j e^{x_j - m}\right)$$

其中 $m = \max(x_j)$。

---

### 块间 LSE 更新公式（跨块合并）

**场景**：合并多个块的 LSE 结果。

设：
- $L_{\text{old}}$ = 之前所有块的 LSE
- $m_{\text{cur}}$ = 当前块的最大值
- $s_{\text{cur}}$ = 当前块的指数和（已减去 $m_{\text{cur}}$）

**推导新的 LSE**：

由 LSE 定义，合并后的指数和为：

$$e^{L_{\text{new}}} = e^{L_{\text{old}}} + \underbrace{e^{m_{\text{cur}}} \cdot s_{\text{cur}}}_{\text{当前块的实际指数和}}$$

因此：

$$L_{\text{new}} = \log\left(e^{L_{\text{old}}} + e^{m_{\text{cur}}} \cdot s_{\text{cur}}\right)$$

**数值稳定性处理**：提取 $m_{\text{cur}}$

$$L_{\text{new}} = m_{\text{cur}} + \log\left(e^{L_{\text{old}} - m_{\text{cur}}} + s_{\text{cur}}\right)$$

**这里需要一个教学上的限定**：
- 本文档与 puzzle 参考实现一致，直接使用当前块最大值 $m_{\text{cur}}$ 作为合并时的锚点
- 当当前块包含新的全局最大值时，$L_{\text{old}} - m_{\text{cur}}$ 通常较小，这个写法直观且与代码一致
- 但严格来说，$L_{\text{old}} - m_{\text{cur}} \leq 0$ **并不总成立**；若历史块的最大值远大于当前块最大值，这一项可能为正
- 因此这个 LSE 写法更适合作为与参考实现一致的教学版本；若追求最稳健的工程实现，可以参考后文的 FlashAttention 风格写法

**具体例子**：

```
假设：
  - 已处理 Block 0-1，L_old = 5
  - Block 2 的 m_cur = 10，s_cur = 3

直接计算：
  L_new = log(e^5 + e^10 × 3) = log(148.4 + 66079.4) ≈ 11.1

稳定计算：
  L_new = 10 + log(e^(5-10) + 3) = 10 + log(0.0067 + 3) = 10 + 1.1 = 11.1

结果相同，但后者避免了例如 e^100 这种潜在溢出风险！
```

### 转换为 log2 空间

GPU 硬件通常对 2 的幂次运算有更直接的支持，因此在教学和工程实践里常优先尝试 `exp2`。

$$\exp(x) = 2^{x \cdot \log_2(e) }$$

块间更新公式转换为 log2 空间：

设 $L' = L \cdot \log_2(e)$，则与参考实现一致的写法可以写成：

$$L'_{\text{new}} = m_{\text{cur}} \cdot \log_2(e) + \log_2\left(2^{L'_{\text{old}} - m_{\text{cur}} \cdot \log_2(e)} + s_{\text{cur}}\right)$$

其中 $\log_2(e) \approx 1.44269504$。

```python
log2_e = 1.44269504
result = T.exp2(x * log2_e)  # 推荐使用
```
