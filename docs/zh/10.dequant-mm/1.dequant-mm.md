# Puzzle 10: Dequantized Matrix Multiplication 代码解析

## 概述

这个 puzzle 介绍了深度学习模型部署中的关键技术：**反量化矩阵乘法（Dequantized Matrix Multiplication）**。
为了教学清晰，本文先从一个最简单的版本开始：把两个 4-bit 权重打包进一个字节，解包后做一个固定偏移的反量化，再接普通矩阵乘法。

## 为什么反量化矩阵乘法如此重要？

在 LLM 部署中，模型量化是降低内存占用和加速推理的关键技术：

- **内存节省**: INT4 量化比 FP16 节省 75% 的显存
- **带宽优化**: 更小的数据传输量，降低内存带宽压力
- **推理加速**: 结合 Tensor Core，可以实现高效推理
- **实际应用**: GPTQ、AWQ、llama.cpp 等量化框架的核心算子

**典型应用场景**:
- GPTQ 量化模型推理
- AWQ 量化模型推理
- llama.cpp INT4 量化
- vLLM 量化推理

> 但要注意：真实系统里的量化 kernel 往往还会带有 `scale`、`zero-point`、group-wise/per-channel 元数据，甚至会把反量化和矩阵乘法更深地融合。
> 本 puzzle 对齐的是“最基础的 packed-int4 解包 + 偏移还原 + GEMM”教学版本，而不是完整复现这些框架的生产实现。

## 问题定义

### 量化背景

**INT4 量化**: 将 FP16 权重量化到 4-bit 整数，但 4-bit 小于一个字节，通常打包存储。

为了循序渐进，本文把“量化权重如何还原”拆成三个层次理解：

1. **解包（unpack）**
   把一个 `uint8` 里的高 4 位和低 4 位拆开。
2. **符号/偏移还原**
   把 0-15 的 nibble 映射回一个以 0 为中心的小整数范围。
3. **真正的反量化（advanced）**
   在真实量化系统里，通常还要继续乘以 `scale`，有时还要减 `zero-point`，即
   `real_value = scale * (q - zero_point)`。

当前 puzzle 只实现到第 2 层，等价于一个非常简单的特例：`scale = 1`，`zero_point = 8`。

```
原始权重 (FP16): [w0, w1, w2, w3, ...]  每个元素 16 bits
量化后 (INT4):   [w0, w1, w2, w3, ...]  每个元素 4 bits
打包存储 (UINT8): [w0|w1, w2|w3, ...]   每个字节存储 2 个 INT4
```

### 输入输出

**输入**:
- `A: [M, K]` - 矩阵 A (float16)，激活值
- `B: [K, N // 2]` - 矩阵 B (uint8)，打包的 INT4 权重
- `M: int` - 矩阵 A 的行数, 1 ≤ M ≤ 8192
- `N: int` - 矩阵 B 的实际列数, 1 ≤ N ≤ 8192
- `K: int` - K 维度, 1 ≤ K ≤ 8192

**输出**:
- `C: [M, N]` - 输出矩阵 (float16)

**中间变量**:
- `ACC0, ACC1: float32` - 累加器
- `B_high, B_low: float16` - 解包后的 INT4 值

> 当前 puzzle 的参考实现主要关注“解包 + 还原 + GEMM”的主逻辑。
> 为了简化讲解，默认 `M % BLOCK_M == 0`、`N % BLOCK_N == 0`、`K % BLOCK_K == 0`；若要支持任意尺寸，需要额外补齐加载和写回时的边界处理。

### 计算定义

```python
for i in range(M):
    for j in range(N // 2):
        ACC0 = 0
        ACC1 = 0
        for k in range(K):
            # 解包 INT4 值
            B_low = float16(B[k, j] & 0x0F) - 8.0   # 低 4 位
            B_high = float16((B[k, j] >> 4) & 0x0F) - 8.0  # 高 4 位
            # 累加计算
            ACC0 += A[i, k] * B_low
            ACC1 += A[i, k] * B_high
        C[i, j * 2] = ACC0
        C[i, j * 2 + 1] = ACC1
```

## 核心概念

### 1. INT4 打包格式

每个 UINT8 存储两个 INT4 值：

```
UINT8 字节: [7:4] | [3:0]
             高位    低位
           B_high  B_low

示例:
B[k, j] = 0xAB  (二进制: 10101011)
B_low  = 0xAB & 0x0F = 0x0B = 11
B_high = (0xAB >> 4) & 0x0F = 0x0A = 10
```

### 2. 偏移还原到有符号范围

当前 puzzle 里的 nibble 值范围是 `[0, 15]`，通过减去 8，把它还原到 `[-8, 7]`：

```python
# nibble: [0, 15]
# 还原后: [-8, 7]

# 当前 puzzle 的还原公式
decoded_value = nibble - 8

# 示例
0x0 = 0  → 0 - 8 = -8
0x7 = 7  → 7 - 8 = -1
0x8 = 8  → 8 - 8 = 0
0xF = 15 → 15 - 8 = 7
```

这一步在教学上足够直观，但要注意它并不覆盖所有 INT4 编码方式。
例如真实量化框架里，常见的是再配合 `scale`/`zero_point` 做仿射反量化，而不是只做一个固定的 `-8`。

### 3. 按位操作详解

```python
# 提取低 4 位
B_low = B[k, j] & 0x0F
# 0x0F = 00001111 (二进制)
# 与操作保留低 4 位，高 4 位清零

# 提取高 4 位
B_high = (B[k, j] >> 4) & 0x0F
# 先右移 4 位，再与 0x0F 确保只有 4 位
```

## PyTorch 参考实现

```python
def ref_dequant_matmul(A: torch.Tensor, B: torch.Tensor):
    assert len(A.shape) == 2
    assert len(B.shape) == 2
    assert A.shape[1] == B.shape[0]  # K
    assert A.dtype == torch.float16
    assert B.dtype == torch.uint8

    K = A.shape[1]
    N = B.shape[1] * 2  # B.shape[1] == N // 2 because of packing

    # 教学版: 先解包，再做最简单的偏移还原
    B_dequantized = torch.zeros((K, N), dtype=torch.float16, device=B.device)
    B_dequantized[:, ::2] = B[:, :] & 0x0F      # 偶数列：低 4 位
    B_dequantized[:, 1::2] = (B[:, :] >> 4) & 0x0F  # 奇数列：高 4 位
    B_dequantized = B_dequantized.to(torch.float16) - 8.0  # 偏移还原到 [-8, 7]

    # 矩阵乘法
    return torch.matmul(input=A, other=B_dequantized)
```

## 与标准矩阵乘法的对比

| 特性 | 标准 GEMM (FP16×FP16) | 反量化 GEMM (FP16×INT4) |
|------|----------------------|------------------------|
| 权重存储 | FP16 (16 bits) | UINT8 (8 bits, 打包 2 个 INT4) |
| 权重内存 | N×K×2 bytes | N×K/2 bytes (节省 75%) |
| 计算步骤 | 直接矩阵乘法 | 解包 → 反量化 → 矩阵乘法 |
| 额外开销 | 无 | 反量化计算 |
| 适用场景 | 训练、推理 | 量化模型推理 |

## 从教学版到真实量化 kernel

当前 puzzle 可以视为下面这个更通用公式的最简特例：

```python
real_weight = scale * (q - zero_point)
```

在本文中：
- `q` 来自解包后的 4-bit 值
- `zero_point = 8`
- `scale = 1`

如果继续往工程实现推进，常见的下一步是：
- 为每个 channel 或每个 group 增加 `scale`
- 让 `zero_point` 也变成 per-group 元数据
- 把这些 metadata 和解包逻辑一起融合进 matmul 主循环

也就是说，本文的 kernel 不是终点，而是理解真实 GPTQ/AWQ 类 kernel 的第一步。

## 性能考量

### 1. 内存带宽优势

```
FP16 权重带宽: M × K × 2 bytes
INT4 权重带宽: M × K × 0.5 bytes  (减少 75%)

即使加上反量化开销，总体仍可提升性能
```

### 2. 反量化开销

```python
# 反量化需要额外的计算
```
