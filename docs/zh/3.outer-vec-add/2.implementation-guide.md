# Outer Vector Add 实现指南

## 任务: 实现 tl_outer_add

### 基础实现（直接访问全局内存）

```python
@tilelang.jit
def tl_outer_add(A, B, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float16
    A: T.Tensor((N,), dtype)
    B: T.Tensor((M,), dtype)
    C = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (pid_n, pid_m):
        n_idx = pid_n * BLOCK_N
        m_idx = pid_m * BLOCK_M

        for i in T.Parallel(BLOCK_N):
            for j in T.Parallel(BLOCK_M):
                C[n_idx + i, m_idx + j] = A[n_idx + i] + B[m_idx + j]

    return C
```

### 关键点

1. **二维 Kernel 启动**
   ```python
   with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (pid_n, pid_m):
   ```
   - 第一个参数: N 方向的 block 数量
   - 第二个参数: M 方向的 block 数量
   - `(pid_n, pid_m)`: 当前 block 在二维 grid 中的坐标

2. **双重并行循环**
   ```python
   for i in T.Parallel(BLOCK_N):
       for j in T.Parallel(BLOCK_M):
   ```
   - 嵌套的 `T.Parallel` 实现二维并行
   - 每个 block 处理 `BLOCK_N × BLOCK_M` 个输出元素

3. **索引计算**
   ```python
   n_idx = pid_n * BLOCK_N  # 当前 block 在 N 维度的起始索引
   m_idx = pid_m * BLOCK_M  # 当前 block 在 M 维度的起始索引
   ```

## 优化版本（使用 Fragment 缓存）

基础版本每次计算都从全局内存读取 `A` 和 `B`，存在重复访问。优化版本使用 `T.copy` 将数据缓存到寄存器：

```python
@tilelang.jit
def tl_outer_add(A, B, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float16
    A: T.Tensor((N,), dtype)
    B: T.Tensor((M,), dtype)
    C = T.empty((N, M), dtype)

    with T.Kernel(N // BLOCK_N, M // BLOCK_M, threads=256) as (pid_n, pid_m):
        n_idx = pid_n * BLOCK_N
        m_idx = pid_m * BLOCK_M

        # 分配寄存器片段
        A_local = T.alloc_fragment((BLOCK_N,), dtype)
        B_local = T.alloc_fragment((BLOCK_M,), dtype)
        C_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)

        # 使用 T.copy 加载数据到寄存器
        T.copy(A[n_idx], A_local)
        T.copy(B[m_idx], B_local)

        # 在寄存器中计算
        for i, j in T.Parallel(BLOCK_N, BLOCK_M):
            C_local[i, j] = A_local[i] + B_local[j]

        # 写回全局内存
        T.copy(C_local, C[n_idx, m_idx])

    return C
```

### 优化说明

| 对比项 | 基础版本 | 优化版本 |
|--------|----------|----------|
| 数据访问 | 每次从全局内存读取 | 预加载到寄存器 |
| 内存访问次数 | `BLOCK_N × BLOCK_M × 2` 次 | `BLOCK_N + BLOCK_M` 次加载 |
| A[i] 重用 | 每个 j 都重新读取 | 缓存后重用 `BLOCK_M` 次 |
| B[j] 重用 | 每个 i 都重新读取 | 缓存后重用 `BLOCK_N` 次 |

### 数据重用图解

```
基础版本:
  for i in range(BLOCK_N):       # A[n_idx + i] 读取 BLOCK_M 次
      for j in range(BLOCK_M):   # B[m_idx + j] 读取 BLOCK_N 次
          C[...] = A[...] + B[...]

优化版本:
  T.copy(A[n_idx], A_local)      # A 只读取 1 次
  T.copy(B[m_idx], B_local)      # B 只读取 1 次
  for i, j in Parallel:          # 全部在寄存器中完成
      C_local[...] = A_local[...] + B_local[...]
```

## API 总结

| API | 说明 |
|-----|------|
| `T.Kernel(grid_x, grid_y, threads=N)` | 二维 grid 启动 |
| `(pid_n, pid_m)` | 二维 block 索引 |
| `T.Parallel(N)` | 并行循环（嵌套使用） |
| `T.Parallel(N, M)` | 二维并行循环（合并写法） |
| `T.alloc_fragment((N, M), dtype)` | 分配二维寄存器片段 |
| `T.copy(src, dst)` | 内存复制 |
