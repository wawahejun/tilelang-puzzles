# Scalar FlashAttention 实现指南

## 核心思路

Scalar FlashAttention = Softmax + 额外计算：

1. **预处理**: 计算 `QK = Q * K`
2. **Softmax**: 对 `QK` 应用 Online Softmax（使用 LSE）
3. **后处理**: 将 softmax 结果乘以 `V`

关键是将这三步融合，避免存储中间结果。

---

## 完整实现

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_scalar_flash_attn(Q, K, V, BLOCK_B: int, BLOCK_S: int):
    log2_e = 1.44269504
    B, S = T.const("B, S")
    dtype = T.float32
    Q: T.Tensor((B, S), dtype)
    K: T.Tensor((B, S), dtype)
    V: T.Tensor((B, S), dtype)
    O = T.empty((B, S), dtype)

    with T.Kernel(B // BLOCK_B, threads=256) as pid_b:
        # 分配 Fragment
        Q_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        K_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        V_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        O_local = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        
        # 当前块的中间结果
        cur_QK = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        cur_exp_QK = T.alloc_fragment((BLOCK_B, BLOCK_S), dtype)
        cur_max_QK = T.alloc_fragment((BLOCK_B,), dtype)
        cur_sum_exp_QK = T.alloc_fragment((BLOCK_B,), dtype)
        
        # LSE: 跨所有块累积的结果
        lse = T.alloc_fragment((BLOCK_B,), dtype)
        T.fill(lse, -T.infinity(dtype))
        
        # 第一次遍历: 计算 LSE（Online Softmax）
        for s_blk_id in T.Serial(S // BLOCK_S):
            # 加载当前块
            T.copy(Q[pid_b * BLOCK_B, s_blk_id * BLOCK_S], Q_local)
            T.copy(K[pid_b * BLOCK_B, s_blk_id * BLOCK_S], K_local)
            
            # 计算 Q * K
            for i, j in T.Parallel(BLOCK_B, BLOCK_S):
                cur_QK[i, j] = Q_local[i, j] * K_local[i, j]
            
            # 找最大值
            T.reduce_max(cur_QK, cur_max_QK, dim=1, clear=True)
            
            # 计算 exp(QK - max)
            for i, j in T.Parallel(BLOCK_B, BLOCK_S):
                cur_exp_QK[i, j] = T.exp2(cur_QK[i, j] * log2_e - cur_max_QK[i] * log2_e)
            
            # 求和
            T.reduce_sum(cur_exp_QK, cur_sum_exp_QK, dim=1, clear=True)
            
            # 更新 LSE
            for i in T.Parallel(BLOCK_B):
                lse[i] = cur_max_QK[i] * log2_e + T.log2(
                    T.exp2(lse[i] - cur_max_QK[i] * log2_e) + cur_sum_exp_QK[i]
                )
        
        # 第二次遍历: 计算最终输出
        for s_blk_id in T.Serial(S // BLOCK_S):
            T.copy(Q[pid_b * BLOCK_B, s_blk_id * BLOCK_S], Q_local)
            T.copy(K[pid_b * BLOCK_B, s_blk_id * BLOCK_S], K_local)
            T.copy(V[pid_b * BLOCK_B, s_blk_id * BLOCK_S], V_local)
            
            # 计算 O = softmax(Q*K) * V = exp(Q*K - LSE) * V
            for i, j in T.Parallel(BLOCK_B, BLOCK_S):
                O_local[i, j] = (
                    T.exp2(Q_local[i, j] * K_local[i, j] * log2_e - lse[i]) * V_local[i, j]
                )
            
            T.copy(O_local, O[pid_b * BLOCK_B, s_blk_id * BLOCK_S])

    return O
```

## 关键技术详解

### 1. 与 Softmax 的对比

| 步骤 | Softmax | Scalar FlashAttention |
|------|---------|----------------------|
| 输入加载 | A | Q, K |
| 预处理 | - | QK = Q * K |
| 找最大值 | max(A) | max(QK) |
| 计算 exp | exp(A - max) | exp(QK - max) |
| 求和 | sum(exp) | sum(exp) |
| 更新 LSE | ✓ | ✓ |
| 归一化 | / sum | / sum * V |
| 输出 | softmax(A) | softmax(QK) * V |

**核心区别**:
1. 第一次遍历多了 `Q * K` 计算
2. 第二次遍历多了乘以 `V` 的操作

### 2. 第一次遍历：计算 LSE

```python
for s_blk_id in T.Serial(S // BLOCK_S):
    # 1. 加载 Q 和 K
    T.copy(Q[...], Q_local)
    T.copy(K[...], K_local)
    
    # 2. 计算 Q * K（新增步骤）
    for i, j in T.Parallel(BLOCK_B, BLOCK_S):
        cur_QK[i, j] = Q_local[i, j] * K_local[i, j]
    
    # 3-6. 与 Softmax 相同：max, exp, sum, 更新 LSE
    T.reduce_max(cur_QK, cur_max_QK, dim=1, clear=True)
    ...
```

**关键点**:
- 计算 `Q * K` 后，后续步骤与 Softmax 完全相同
- 使用 `cur_QK` 存储 `Q * K` 的结果
- LSE 更新公式与 Softmax 一致

### 3. 第二次遍历：计算输出

```python
for s_blk_id in T.Serial(S // BLOCK_S):
    # 1. 加载 Q, K, V（新增 V）
    T.copy(Q[...], Q_local)
    T.copy(K[...], K_local)
    T.copy(V[...], V_local)  # 新增
    
    # 2. 计算 O = exp(Q*K - LSE) * V
    for i, j in T.Parallel(BLOCK_B, BLOCK_S):
        O_local[i, j] = (
            T.exp2(Q_local[i, j] * K_local[i, j] * log2_e - lse[i])  # softmax
            * V_local[i, j]  # 乘以 V（新增）
        )
    
    # 3. 写回输出
    T.copy(O_local, O[...])
```

**关键点**:
- 需要加载 V
- 在计算 softmax 的同时乘以 V
- 融合了两个操作：`softmax(QK)` 和 `* V`

### 4. 融合计算的优势

**未融合版本**:
```python
# 需要存储中间结果
QK = Q * K              # 存储 QK[B, S]
P = softmax(QK)         # 存储 P[B, S]
O = P * V               # 存储 O[B, S]
# 总内存: 3 × B × S
```

**融合版本**:
```python
# 只存储最终输出
for each block:
    QK_block = Q_block * K_block  # 临时，在 Fragment 中
    P_block = softmax(QK_block)   # 临时，在 Fragment 中
    O_block = P_block * V_block   # 写回全局内存
# 总内存: B × S + B × BLOCK_S（临时）
```

## 代码结构总结

```
Kernel(B // BLOCK_B)
├── 分配 Fragment
│   ├── Q_local, K_local, V_local, O_local: 存储当前块数据
│   ├── cur_QK: Q * K 的结果
│   ├── cur_exp_QK, cur_max_QK, cur_sum_exp_QK: 当前块中间结果
│   └── lse: 跨所有块累积的 LSE
│
├── 第一次遍历 (计算 LSE)
│   └── for s_blk_id in T.Serial(S // BLOCK_S):
│       ├── 加载 Q_local, K_local
│       ├── 计算 cur_QK = Q * K
│       ├── T.reduce_max 找最大值
│       ├── 计算 exp(QK - max)
│       ├── T.reduce_sum 求和
│       └── 更新 LSE
│
└── 第二次遍历 (计算输出)
    └── for s_blk_id in T.Serial(S // BLOCK_S):
        ├── 加载 Q_local, K_local, V_local
        ├── 计算 O = exp(Q*K - LSE) * V
        └── 写回 O_local
```

## 与 Softmax 实现的对比

### Softmax 实现

```python
# 第一次遍历
T.copy(A[...], A_local)
T.reduce_max(A_local, cur_max, dim=1, clear=True)
# 计算 exp 和更新 LSE
...

# 第二次遍历
T.copy(A[...], A_local)
B_local[i, j] = T.exp2(A_local[i, j] * log2_e - lse[i])
T.copy(B_local, B[...])
```

### Scalar FlashAttention 实现

```python
# 第一次遍历
T.copy(Q[...], Q_local)
T.copy(K[...], K_local)
cur_QK[i, j] = Q_local[i, j] * K_local[i, j]  # 新增
T.reduce_max(cur_QK, cur_max, dim=1, clear=True)
# 计算 exp 和更新 LSE（相同）
...

# 第二次遍历
T.copy(Q[...], Q_local)
T.copy(K[...], K_local)
T.copy(V[...], V_local)  # 新增
O_local[i, j] = T.exp2(Q_local[i, j] * K_local[i, j] * log2_e - lse[i]) * V_local[i, j]  # 融合
T.copy(O_local, O[...])
```

**差异总结**:
1. 输入从 1 个变成 3 个（Q, K, V）
2. 第一次遍历多了 `Q * K` 计算
3. 第二次遍历多了乘以 `V` 的操作

## 性能分析

### 内存访问

| 操作 | 内存读取 | 内存写入 |
|------|---------|---------|
| 第一次遍历 | 2 × B × S (Q, K) | 0 |
| 第二次遍历 | 3 × B × S (Q, K, V) | 1 × B × S (O) |
| 总计 | 5 × B × S | 1 × B × S |

**对比标准实现**:
- 标准: 读 3 × B × S，写 3 × B × S（QK, P, O）
- FlashAttention: 读 5 × B × S，写 1 × B × S
- 写入减少 67%，但读取增加（因为需要重新读取 Q, K）

### 计算复杂度

- Q * K: B × S 次乘法
- Softmax: B × S 次 exp, max, sum
- 乘以 V: B × S 次乘法
- 总计: O(B × S)
