# Puzzle 07: Scalar FlashAttention 代码解析

## 概述

这个 puzzle 介绍如何在 TileLang 中实现 **Scalar FlashAttention**，这是从 Softmax 到完整 FlashAttention 的重要过渡。通过简化版本，我们可以理解 FlashAttention 的核心思想。

## 什么是 FlashAttention?

FlashAttention 是一种高效的注意力机制实现，通过减少内存访问来加速计算。标准注意力机制需要存储完整的注意力矩阵，而 FlashAttention 使用分块计算和 Online Softmax 避免这一开销。

### 标准注意力机制

$$\text{Attention}(Q, K, V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$$

对于完整的注意力：
- $Q, K, V \in \mathbb{R}^{B \times S \times d}$（B: batch size, S: sequence length, d: head dimension）
- 需要存储 $QK^T \in \mathbb{R}^{B \times S \times S}$，内存开销 $O(S^2)$

### Scalar FlashAttention 简化

为了渐进式学习，我们做了以下简化：

1. **移除多头注意力**: 只有单个头
2. **标量版本**: $Q, K, V \in \mathbb{R}^{B \times S}$，逐元素相乘而不是矩阵乘法
3. **移除缩放因子**: 不除以 $\sqrt{d_k}$

简化后的计算：

$$O = \text{softmax}(Q \odot K) \odot V$$

其中 $\odot$ 表示逐元素乘法。

## 问题定义

### 输入
- `Q: [B, S]` - Query 张量，float32
- `K: [B, S]` - Key 张量，float32
- `V: [B, S]` - Value 张量，float32
- `B: int` - Batch size，1 ≤ B ≤ 256
- `S: int` - Sequence length，1 ≤ S ≤ 16384

### 输出
- `O: [B, S]` - 输出张量，float32

### 中间变量
- `QK: [B, S]` - Q 和 K 的逐元素乘积
- `P: [B, S]` - Softmax 结果（归一化前）
- `MAX: float32` - 每行的最大值
- `SUM: float32` - 每行的指数和

### 计算流程

```python
for i in range(B):
    # 步骤1: 计算 Q*K 并找最大值
    MAX = -inf
    for j in range(S):
        QK[i, j] = Q[i, j] * K[i, j]
        MAX = max(QK[i, j], MAX)
    
    # 步骤2: 计算 exp 和求和
    SUM = 0
    for j in range(S):
        P[i, j] = exp(QK[i, j] - MAX)
        SUM += P[i, j]
    
    # 步骤3: 归一化并乘以 V
    for j in range(S):
        O[i, j] = P[i, j] / SUM * V[i, j]
```

## PyTorch 参考实现

```python
def ref_scalar_flash_attn(Q: torch.Tensor, K: torch.Tensor, V: torch.Tensor):
    return torch.softmax(Q * K, dim=1).mul_(V)
```

非常简洁：先计算 `Q * K`，然后 softmax，最后乘以 `V`。

## 与 Softmax 的关系

Scalar FlashAttention 本质上是 **Softmax + 额外计算**：

| 步骤 | Softmax | Scalar FlashAttention |
|------|---------|----------------------|
| 输入 | A | Q, K, V |
| 预处理 | - | QK = Q * K |
| Softmax | softmax(A) | softmax(QK) |
| 后处理 | - | 结果 * V |

因此，我们可以直接复用 Softmax 的实现，只需：
1. 在第一次遍历中计算 `Q * K`
2. 在第二次遍历中将 softmax 结果乘以 `V`

## 核心思想

### 1. 融合计算

标准实现需要 4 个步骤：
```python
QK = Q * K          # 步骤1
P = softmax(QK)     # 步骤2（包含 max, exp, sum）
O = P * V           # 步骤3
```

FlashAttention 将这些步骤融合，避免存储中间结果 `QK` 和 `P`。

### 2. Online Softmax

使用 LSE (log-sum-exp) 方法，在 S 维度分块处理：
- 第一次遍历：计算 `Q * K`，同时累积 LSE
- 第二次遍历：使用 LSE 计算最终输出 `softmax(QK) * V`

### 3. 内存效率

**标准实现**:
```
内存: QK[B, S] + P[B, S] + O[B, S] = 3 × B × S
```

**FlashAttention**:
```
内存: O[B, S] + 小块缓存 = B × S + B × BLOCK_S
```

对于 S=16384，内存节省显著。

## 算法流程

```
输入: Q[B, S], K[B, S], V[B, S]
  ↓
┌─────────────────────────────────┐
│ 对每个 batch (并行处理 BLOCK_B)  │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 第一次遍历 (计算 LSE)            │
│ for s_blk in Serial(S // BLOCK_S): │
│   1. 加载 Q, K 的当前块          │
│   2. 计算 QK = Q * K             │
│   3. 找最大值 max(QK)            │
│   4. 计算 exp(QK - max)          │
│   5. 求和                        │
│   6. 更新 LSE                    │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│ 第二次遍历 (计算输出)            │
│ for s_blk in Serial(S // BLOCK_S): │
│   1. 加载 Q, K, V 的当前块       │
│   2. 计算 O = exp(Q*K - LSE) * V │
│   3. 写回输出                    │
└─────────────────────────────────┘
  ↓
输出: O[B, S]
```

## 与完整 FlashAttention 的区别

| 特性 | Scalar FlashAttention | 完整 FlashAttention |
|------|----------------------|-------------------|
| 维度 | Q, K, V: [B, S] | Q, K, V: [B, S, d] |
| 计算 | Q * K (逐元素) | Q @ K^T (矩阵乘法) |
| 注意力矩阵 | [B, S] | [B, S, S] |
| 多头 | 无 | 有 |
| 缩放 | 无 | 除以 √d |
| 内存节省 | 中等 | 显著 (O(S²) → O(S)) |

## 学习要点

1. **融合计算**: 将多个操作融合到一个 kernel 中
2. **Online Softmax**: 复用 Softmax 的 LSE 技术
3. **分块处理**: 在 S 维度串行分块，避免存储完整矩阵
4. **内存效率**: 只存储最终输出，不存储中间结果
5. **渐进式学习**: 从简化版本理解核心思想，再扩展到完整版本

## 性能考虑

Scalar FlashAttention 是 **计算和内存双重受限**：
- 计算: Q*K, exp, 除法, 乘以 V
- 内存: 需要多次读取 Q, K, V

优化策略：
1. 使用 Online Softmax 减少遍历次数
2. 分块处理适应内存限制
3. 融合计算减少中间结果存储
4. 批量内存传输提高带宽利用率
