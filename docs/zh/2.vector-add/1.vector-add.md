# Puzzle 02: Vector Add 代码解析

## 概述

这个puzzle介绍了如何在TileLang中实现向量加法操作，并逐步展示了GPU编程中的优化技术。

## 核心概念

### 1. 基础向量加法 (tl_add_1d)

**目标**: 实现 `C[i] = A[i] + B[i]`

```python
@tilelang.jit
def tl_add_1d(A, B, BLOCK_N: int):
    N = T.const("N")
    A: T.Tensor((N,), T.float16)
    B: T.Tensor((N,), T.float16)
    C = T.empty((N,), T.float16)

    with T.Kernel(N // BLOCK_N, threads=256) as bx:
        base_idx = bx * BLOCK_N
        for i in T.Parallel(BLOCK_N):
            C[base_idx + i] = A[base_idx + i] + B[base_idx + i]

    return C
```

**关键点**:
- `T.Kernel(N // BLOCK_N, threads=256)`: 创建 N//BLOCK_N 个block，每个block有256个线程
- `bx`: 当前block的索引
- `T.Parallel(BLOCK_N)`: 并行循环，将BLOCK_N个元素分配给256个线程处理
- 元素级操作: `A[i] + B[i]` 是标量加法，不是TileOps

### 3. 内存优化版本 (tl_mul_relu_1d_mem)

**优化目标**: 减少全局内存访问，使用寄存器优化

**GPU内存层次**:
- **全局内存 (DRAM)**: 最慢，容量最大
- **共享内存 (Shared Memory)**: 中等速度
- **寄存器 (Registers)**: 最快，容量最小

**优化策略**:
1. 使用 `T.alloc_fragment` 分配寄存器片段
2. 批量加载数据到寄存器 (如使用ldg128一次加载128位)
3. 在寄存器中进行计算
4. 批量写回结果

**Fragment概念**:
- Fragment是TileLang对寄存器的抽象
- 在CUDA中，寄存器是线程局部的
- TileLang的Fragment代表一个block中所有线程的寄存器集合
- 可以像操作Buffer一样统一操作Fragment

预期优化效果:
- 减少全局内存访问次数

## 学习要点

1. **循环抽象**: `T.Parallel` 用于并行迭代
2. **元素级操作**: 加法、乘法等是标量操作，不是TileOps
3. **条件逻辑**: `T.if_then_else` 实现条件选择
4. **内存优化**: Fragment抽象简化了寄存器管理
5. **算子融合**: 将多个操作融合到一个kernel中减少内存访问
