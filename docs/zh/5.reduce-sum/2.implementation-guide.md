# Reduce Sum 实现指南

## 核心思路

Reduce Sum 需要对每一行的所有元素求和：

```python
for i in range(N):
    B[i] = sum(A[i, :])  # 对第 i 行求和
```

关键挑战：
1. M 维度太大，无法一次加载全部
2. 需要分块处理并累积结果
3. 使用 `T.reduce_sum` 高效归约

---

## 基础版本（手动累加）

使用逐元素累加的方式理解归约的本质：

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_reduce_sum_basic(A, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N,), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as bx:
        n_idx = bx * BLOCK_N
        
        # 分配 Fragment
        A_frag = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_frag = T.alloc_fragment((BLOCK_N,), dtype)
        
        # 初始化为 0
        for i in T.Parallel(BLOCK_N):
            B_frag[i] = T.float32(0)
        
        # 串行遍历 M 维度的所有块
        for m_start in T.Serial(M // BLOCK_M):
            m_idx = m_start * BLOCK_M
            
            # 加载当前块
            T.copy(A[n_idx:n_idx+BLOCK_N, m_idx:m_idx+BLOCK_M], A_frag)
            
            # 逐行累加
            for i in T.Parallel(BLOCK_N):
                # 使用 T.alloc_var 创建可变变量
                row_sum = T.alloc_var(dtype)
                row_sum = T.float32(0)
                for j in T.serial(BLOCK_M):
                    row_sum += A_frag[i, j]
                B_frag[i] += row_sum
        
        # 写回全局内存
        T.copy(B_frag, B[n_idx:n_idx+BLOCK_N])

    return B
```

**特点**：
- 使用嵌套循环手动累加
- 使用 `T.alloc_var` 创建可变变量
- 易于理解归约的本质
- 但效率较低

## 优化版本（使用 T.reduce_sum）

使用 TileLang 提供的高级归约 API：

```python
@tilelang.jit(
    pass_configs={
        tilelang.PassConfigKey.TL_DISABLE_WARP_SPECIALIZED: True,
        tilelang.PassConfigKey.TL_DISABLE_TMA_LOWER: True,
    },
)
def tl_reduce_sum(A, BLOCK_N: int, BLOCK_M: int):
    N, M = T.const("N, M")
    dtype = T.float32
    A: T.Tensor((N, M), dtype)
    B = T.empty((N,), dtype)

    with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
        # 分配 Fragment
        A_local = T.alloc_fragment((BLOCK_N, BLOCK_M), dtype)
        B_local = T.alloc_fragment((BLOCK_N,), dtype)
        
        # 初始化为 0
        T.clear(B_local)
        
        # 串行遍历 M 维度的所有块
        for m_blk_id in T.Serial(M // BLOCK_M):
            # 加载当前块
            T.copy(A[pid_n * BLOCK_N, m_blk_id * BLOCK_M], A_local)
            
            # 归约求和并累积到 B_local
            T.reduce_sum(A_local, B_local, dim=1, clear=False)
        
        # 写回全局内存
        T.copy(B_local, B[pid_n * BLOCK_N])

    return B
```


---

## 关键技术详解

### 1. Kernel 配置

```python
with T.Kernel(N // BLOCK_N, threads=256) as pid_n:
```

- 只在 N 维度分块
- 每个 block 处理 `BLOCK_N` 行
- 总共 `N // BLOCK_N` 个 block

### 2. 初始化

```python
T.clear(B_local)
```

- 使用 `T.clear` 将 `B_local` 初始化为 0
- 比手动循环初始化更简洁

### 3. 串行循环遍历 M 维度

```python
for m_blk_id in T.Serial(M // BLOCK_M):
    # 处理第 m_blk_id 个块
```

**为什么用 `T.Serial`？**
- M 维度太大，无法一次加载全部
- 需要分块处理并累积结果
- 串行保证正确的累加顺序

### 4. T.reduce_sum 的关键参数

```python
T.reduce_sum(A_local, B_local, dim=1, clear=False)
```

**参数说明**:
- `A_local`: 输入 Fragment
- `B_local`: 输出 Fragment（累积结果）
- `dim=1`: 沿第二维（列）归约
- `clear=False`: **不清零输出**，累积到现有值

**为什么 `clear=False`？**

```python
# 第一次迭代
T.reduce_sum(A_local, B_local, dim=1, clear=False)
# B_local[i] = 0 + sum(A_local[i, :])

# 第二次迭代
T.reduce_sum(A_local, B_local, dim=1, clear=False)
# B_local[i] = B_local[i] + sum(A_local[i, :])  # 累积

# 第三次迭代...
```

如果 `clear=True`，每次都会清零 `B_local`，丢失之前的累积结果。

**内存访问模式**

```
全局内存 A[N, M]
    ↓ T.copy (批量加载 BLOCK_M 列)
Fragment A_local[BLOCK_N, BLOCK_M]
    ↓ T.reduce_sum (沿 dim=1 归约)
Fragment B_local[BLOCK_N] (累积)
    ↓ T.copy (批量写回)
全局内存 B[N]
```

## 代码结构总结

```
Kernel(N // BLOCK_N)
├── 分配 Fragment
│   ├── A_local[BLOCK_N, BLOCK_M]: 输入块
│   └── B_local[BLOCK_N]: 累积和
│
├── 初始化
│   └── T.clear(B_local)  # 清零
│
├── 串行遍历 M 维度
│   └── for m_blk_id in T.Serial(M // BLOCK_M):
│       ├── 加载 A_local
│       └── T.reduce_sum(..., clear=False)  # 累积
│
└── 写回结果
    └── T.copy(B_local, B[...])
```

## 为什么需要串行循环？

### 问题：M 维度太大

```
M = 16384, BLOCK_M = 128
需要 16384 / 128 = 128 次迭代

无法一次加载全部 16384 列到 Fragment
```

### 解决方案：分块累积

```
迭代 0: 加载 A[:, 0:128]     → B += sum(A[:, 0:128])
迭代 1: 加载 A[:, 128:256]   → B += sum(A[:, 128:256])
...
迭代 127: 加载 A[:, 16256:16384] → B += sum(A[:, 16256:16384])

最终: B[i] = sum(A[i, :])
```
